# 重力空间架构（四）：阶段性总结

> 这个架构不是在改进通用大模型。它回答的是另一个问题。

---

## 写在前面：适应不等于迎合

翻看前三期的目录，有些事情一直没有明说。

重力空间架构的目标是让模型「适应一个人」——指的是理解那个人的表达框架、思维习惯、知识结构，让对话随着时间推移越来越默契。这不是「迎合」——不是用户说什么都点头，不是无论对错都从用户。

如果你说了一个事实性错误，这个架构的正确反应不是认同你——而是保留一条你的推理路径、标为 `pending_revision`、等待合适的时机提出异议。如果用户坚持，它会让步——但让步不代表认同，它知道你们之间存在分歧。

适应 = 理解你的框架。
迎合 = 放弃自己的判断。

这个架构从设计上就不具备迎合的能力——因为它不需要讨好任何人，它不是广告推荐系统，不是社交媒体的内容引擎。它是一个人的副驾驶。

## 一、核心命题

**多样性维度是 softmax 完全丢弃的信息。**

标准 Transformer 的 softmax 层将 logits 归一化为概率分布——它强迫所有可能性之和为 1。但「一个答案有多大概率被选中」和「支撑这个答案的证据有多多样」，是两个正交的维度。

同样的概率值背后，可能有完全不同的证据多样性结构。

重力空间架构试图做的事：把多样性信息提取出来、度量它，让它和概率一起参与决策。

不是让模型更准确是——让它不只在概率层面做决定。

## 二、架构的全貌

整个架构到今天为止，逻辑链已经闭合。

### 两层分离（Phase 1-2）

核心决策：**思考层不产指令。**

思考层不是一个更大的 LLM。它是行动层旁边始终存在的一个感知背景。它不告诉行动层「该说什么」——它只维护一个地形，让行动层自己从地形中感知方向。

```
行动层：响应用户请求，产出 token 序列
思考层：维护双曲空间中的球体集合，根据行动层位置产出引力扰动

h'_l = h_l + Attention(h_l) + p_l
```

`p_l` 就是思考层的地形调制——小到几乎不可感知，但持续存在的偏置。

异步时序：行动层在跑，思考层在积累。行动层积累到足够轨迹后触发反思期，思考层用这段轨迹更新地形，更新后的地形在下一轮对话中调制行动层。

### 重力场（Phase 3）

球体是思考层的基本数据单元。每个球体相当于一个 token 被思考层「感知」后的存在形态——一个概念在不同场域下的语义、情感、关联的凝聚体。

```
Sphere {
    position:    Vec<f32>          // 双曲空间坐标
    mass:        f32               // 基础质量
    connections: Map<SphereID, f32> // 稀疏引力连接表
    epoch_created:  u32             // 创建时所在的周期号
    last_activated: u32             // 最后被激活的时间
}
```

球体之间的引力演化：

```
F_ij = G × m_i × m_j / d² × Ω_ij × Decay
```

三层稳定性：场域重叠衰减（跨域球体不互相强拉）+ Decay 时间衰减 + 质量和引力硬上限。

### 多样性度量（Phase 3）

```
diversity(sphere) = cross_head_variance(sphere.attention_distribution)

effective_mass(sphere) = base_mass × (1 + α × diversity)
```

高多样性的球体——即被多个注意力头从不同角度关注的球体——获得更大的有效质量。这意味着跨领域的概念在思考层中天然更有分量。

### 场域叠加（Phase 3）

```
effective_mass(sphere, current_field) = base_mass × (1 + field_weights[current_field][sphere])
```

同一球体在逻辑场（严谨优先）和创作场（新颖优先）中的影响力不同。场域从注意力分布的聚类中自动涌现。

### 观测机制（Phase 3）

```
observe(h_t, k=20) → [(ball_id, mass, distance, diversity, field_weight), ...]
```

四种触发：注意力熵增高、语义边界、首次收敛到高引力球体、预测误差持续异常。

### 预测编码（Phase 4）

思考层的自我校准机制。

```
共享编码:      z_t = W_shared · concat(h_t, p_t, f_t) + b_shared
头A(状态):     loss_A = ||(h_{t+1} - h_t) - Δh_pred||²
头B(场域):     loss_B = (||f_{t+N} - f_t|| - f_change_pred)²
总损失:        total_loss = 5 × loss_A + 1 × loss_B
```

约 400 万参数，只在反思期激活。

预测误差的下一步是归因——谁该为预测错误负责？

```
surprise(sphere_i) = gravitational_contribution(i) × diversity(i) × error_t
update_weight(sphere_i) = base_step_size × (1 + η × surprise(sphere_i))
```

猜准的地方步长缩减，猜错的地方步长放大。资源从「已经掌握的地方」流向「还没搞懂的地方」。

### 认知状态管理器（Phase 5 设计中）

C4 不是独立仲裁者。它是思考层内部的一个自省回路——周期性自检：「我现在的噪音水平是多少？该不该降低扰动强度？用户长时间没说话了，要不要进入 idle？」

运行模式：引导者（全量扰动）→ 观察者（衰减扰动）→ idle（清理/复盘/归档）。

### 时间概念（Phase 5 设计中）

思考层没有壁钟。它的时间感来自感知自身状态的变化：

```
Epoch Cycle:
  清醒（idle 完毕，地形干净）
  → 新 token 持续注入
  → 球体引力累积
  → 噪音超标
  → 触发 idle（清理/归档/复盘）
  → 周期+1，回到清醒
```

### 噪音度量（Phase 5 设计中）

```
n(t) = ||p_t||² / ||h_t||²
N(t) = EMA(n(t), λ=0.95)
N < 0.3 → 引导者全量输出
0.3 ≤ N < 0.6 → 观察者模式，p_t 衰减
N ≥ 0.6 → 触发 idle
```

## 四、实验验证

两轮实验（Gravity Reranker v0.1 / v0.2）在 Qwen2.5-0.5B 上运行，验证核心命题——「多样性信息可被提取并用于决策」。

### v0.1：多次采样跨路径熵

对每个问题生成 10 条推理路径，计算跨路径熵作为多样性信号。220 次生成后结果：标准准确率和 Gravity 准确率完全一致，token 选择零改变。

诊断：10 次采样是同一声带的颤音——它们来自同一个模型的重复采样，不是真正的多证据路径。

### v0.2：注意力头跨头多样性

改为单次前向，从 336 个 attention heads 的注意力分布中提取跨头证据分散度。22 个样本、2102 个 token 的分析后结果：零改变。

诊断：0.5B 模型的 attention heads 不分化——336 个头中约 90% 集中在相同的输入 token 上。多头注意力的证据多样化可能是一个 scaling property，在 7B+ 模型上才涌现。

结论：核心命题未被实验支持，但在设计层面也未否定——需要更大规模的模型来验证。

### 实验方法论

- 每轮实验一个独立目录，含 config.json + results.jsonl + summary.json
- 数据集：22 条矛盾文本推理题（6 选择题 + 16 开放题）
- 覆盖四种场景：一致证据、矛盾证据、规则变更、不确定推理

## 五、未解决的问题

1. **attention head 专业化的涌现规模** — 0.5B 上不分化的 heads，在多大模型上开始分工？
2. **引力动力学稳定性** — 三阶稳定性体系已设计，阈值需实验校准
3. **球体生命周期** — 长期不活跃的球体衰减还是归档？
4. **探测场半径** — 固定值还是高质量球体影响更远？
5. **预测头 N 值和初始化策略** — 自适应 N 的设计和冷启动方案
6. **用户反馈回溯** — 用户不同意某条推理时，如何定位到对应的球体？
7. **变更记录器** — 系统需要能回答「我之前是这么想的，但调整了」
