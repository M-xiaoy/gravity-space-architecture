**用途：** 行动层每层调用（或按成本策略选择性调用），获取思考层的地形调制。

- 深层（后 1/3）每层查

## 需要定义的接口（Phase 5 设计中）

### C4 → C1/C2/C5（状态管理器 → 各组件）

```
set_mode(组件: C1|C2|C5, 模式: ONLINE|OBSERVE|IDLE|INTERRUPTED)
```

认知状态管理器向各组件下发模式切换指令。

**待定：** 如果 C4 是自省回路（方案 B），这个接口不是跨组件调用而是内部方法。

---

### C4 → 用户（告警接口）

```
report_status(类型: NOISE_HIGH | IDLE_ENTER | PENDING_REVISION) → 用户响应
```

系统主动向用户报告当前状态。例如：「当前噪音太高，我建议暂停整理思绪。」

---

### C5 → C2（Idle 写入接口）

```
idle_cleanup(球体ID列表) → void       /* 噪音清理 */
idle_replay(轨迹片段) → void           /* 内容复盘 */
idle_archive(截止时间戳) → void        /* 记忆归档 */
```

Idle 线程对重力场的写入操作。三种操作各有不同的写入语义。

---

### C6 → C2（反馈注入接口）

```
inject_feedback(pending_revision: {路径, 影响因素}) → 吸收确认
```

将用户不同意标记得 pending_revision 注入重力场的待吸收区，等待下一次 idle 时处理。

---

### C7（变更记录接口）

```
record_change(组件ID, 旧状态, 新状态, 原因) → void
query_change_history(时间范围 | 组件ID) → 变更列表
dump_self_narrative() → "我之前是这么想的，因为X原因调整了"
```

变更记录器的核心操作。`dump_self_narrative()` 是面向用户的接口——系统描述自己为何改变。
