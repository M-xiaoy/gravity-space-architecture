# 重力空间架构 · Gravity Space Architecture

**一个面向 personal model 的 AI 推理架构提案。**

不是通用大模型的改进方案。是回答一个不同问题的尝试：

> 如果模型只服务于一个人——它的知识来自这个人的文档和对话，它的偏好收敛到一个人的认知模式，它的场域权重由这个人亲手设定——它的架构应该长什么样？

[查看架构工程图 →](docs/engineering-diagram.html)

---

## 核心主张

标准 Transformer 隐含一个假设：**思考和行动在每一步锁步。** 每生成一个 token，模型重新计算一次注意力——这是它的「思考」。但这个思考没有潜伏期，没有权重演化。

重力空间架构提出两个子系统以不同时间尺度运行：

- **行动层** — 标准 Transformer，步频 token 级，负责任务执行
- **思考层** — 双曲空间中的引力模型，步频语义回合级，负责多源证据积累

## 关键概念

- **地形** — 思考层维护的双曲空间，球体（token/概念）在其中分布，位置和质量编码了多源证据的积累
- **场域** — 逻辑、闲谈、查询等认知语境共享同一空间，但同一个球体在不同场域下有不同的有效质量
- **多样性** — 由执行层的注意力分布自动度量，多条独立推理路径同时支持的选项权重更高
- **观测** — 行动层可主动查询思考层的引力分布快照，带来可解释性和自我纠偏能力
- **预测编码** — 思考层对行动层下一步走向产生预期，预期与实际的差额（预测误差）驱动地形更新
- **惊异度归因** — 高多样性球体出错时比低多样性球体分配更多更新资源
