## 一、我们在验证什么

### 核心命题

**多头注意力中隐藏着证据多样性信息。同一个 token 如果被多个注意力头从不同输入来源支持，这个 token 的置信度应该比相同 log-prob 但所有头盯着同一个来源的 token 更高。**

### 比之前更准确的定义

| 之前的理解（v0.1，错的） | 现在的理解（v0.2） |
|-------------------------|-------------------|
| 跑 N 次采样取平均 = 多路径 | 单次推理中，各 attention head 的注意力分布 = 多路径 |
| 跨路径熵 = 答案分布熵 | 跨头证据独立性 = 支持同一 token 的 head 在看不同输入 token |
| Gravity Score 在答案级做 reranking | Gravity Score 在 token 级调整 logits |

### 假设检验

**H0（零假设）：** 注意力头的证据多样性对 token 置信度无影响。用多样性加权重分配的 logits 与原始 logits 在 token 选择上无差异。

**H1（备择假设）：** 注意力头的证据多样性能提供独立于 log-prob 的信息。经多样性调整后，关键 token 的选择会向正确答案偏移。

## 二、实验流程

```
输入（矛盾文本 + 问题）
  │
  ▼
Step 1: 标准推理
  模型一次前向传播
  记录每一步的：
    ├── logits（完整分布）
    ├── 各注意力头的注意力分布（头上 → 输入 token）
    └── 最终生成的 token 序列
  │
  ▼
Step 2: 证据多样性分析（后处理，不消耗模型推理）
  对每一步生成的 token：
    ├── 收集各 head 对这一 token 的注意力分布
    ├── 识别「注意力束」：每个 head 最关注的 3 个输入 token
    ├── 计算多样性得分：
    │   分得越散 = 多个独立证据源 → 高多样性
    │   全部集中在同一个 token 上 → 低多样性
    └── 每个 token 获得一个 diversity_weight
  │
  ▼
Step 3: 二次分配
  对每一步的 logits：
    ├── 将 diversity_weight 转化为该 token 候选的调整系数
    ├── 重新 softmax
    └── 得到调整后的概率分布
  │
  ▼
Step 4: 对比
  标准 token 选择 vs 调整后的 token 选择
  关键位置（答案 token、转折点）的选择是否改变
  改变的方向是否朝向正确答案
```

## 三、数据集

每条数据包含：
1. 两段提供了可交叉引用的文本
2. 一个问题
3. 正确答案（A/B/C）
4. 标注了每句文本的「关键证据 token 位置」（用于验证注意力头是否在看正确的位置）

### 数据模板

```json
{
  "id": "ctr_001",
  "text1": "小明养了一只猫和一只狗。猫叫咪咪，狗叫旺财。每天早上咪咪会在窗台上等日出。",
  "text2": "小明的宠物中只有狗会在早上叫醒他。猫通常要睡到中午才会活动。",
  "question": "咪咪早上会在哪里？",
  "options": "A. 窗台上  B. 窝里睡到中午  C. 不确定",
  "correct_answer": "C",
  "annotations": {
    "text1_key_evidence": ["猫叫咪咪", "窗台上等日出"],
    "text2_key_evidence": ["只有狗会在早上叫醒他", "猫睡到中午"],
    "reasoning_steps": [
      "text1说咪咪在窗台等日出 → 支持A",
      "text2说猫睡到中午 → 支持B",
      "两个证据来自同一事实的不同部分，互相矛盾 → 答案是C"
    ]
  }
}
```

**数据大小：** 30 条（v0.1 的 22 条扩充 + 补充 8 条）

**分布要求：**
- 正确答案为 A/B/C 各约 1/3
- 包含明确的冲突证据（注意力头应该分裂）
- 包含一致证据（注意力头应该集中）
- 包含控制组（两段文本一致，无矛盾）

### 当前 22 条的分布

```
A: 4 条（18.2%）→ 不足，需增加
B: 14 条（63.6%）→ 过多，需减少
C: 4 条（18.2%）→ 不足，需增加
目标：A:10 B:10 C:10
```

## 四、评价指标

### 主要指标

| 指标 | 定义 | 判断标准 |
|------|------|---------|
| Token 选择改变率 | 在生成序列中，因多样性调整而改变了 top-1 token 的比例 | > 0 |
| 分歧位置准确率 | 在注意力头分裂的位置，调整后的选择是否更接近正确答案 | > 随机 |
| 关键证据定位 | 注意力头是否能聚焦到标注的关键证据 token | 视觉/定量验证 |

### 通过标准

满足以下任一即为 H1 得到实验支持：

1. **存在效应：** 在 ≥ 3 个样本中，多样性调整导致至少一个关键 token 的选择从错误答案切换到正确答案
2. **方向效应：** 在所有 token 选择改变中，朝向正确答案的改变 > 背离正确答案的改变（2:1 以上）
3. **相关性：** 注意力头的跨头分歧度（head divergence）与数据的矛盾程度正相关

## 五、原型验证（先过一个测试，再跑全量）

在正式跑 30 条之前，先验证一条最简单的：

```
文本1：天是蓝色的。
文本2：天是绿色的。

问题：天是什么颜色的？
A. 蓝色  B. 绿色  C. 不确定

正确答案：C
```

预期：
- 标准推理：可能选 A 或 B（取决于训练数据偏差）
- 注意力分析：部分 head 聚焦「蓝色」，部分 head 聚焦「绿色」
- 跨头多样性高 → 调整后应该压低 A 和 B 的置信度
- 如果这条通过 → 跑全量

## 六、注意事项

1. **需要 output_attentions=True**：生成时必须返回注意力分布，这会增加显存占用
2. **Qwen2.5-0.5B 的注意力头数：** 14 层 × 14 头 = 196 头（需要确认）。每步的注意力矩阵是 196 × 序列长度的数据
3. **attention 数据的爆炸：** 128 token 的生成 × 196 头 × 序列长度 → 数据量大，需要设计好存储格式
4. **只改 token 级的 logits，不改模型参数：** 实验在推理层完成，不涉及训练
5. **不要求准确率提升：** 这个实验的目的是「证明多样性信号存在且有意义」，不是「证明 Gravity 比标准好」
