## Step 1：配置参数

**文件：** `config.py`

新增一段配置：

```python
@dataclass
class ClusteringConfig:
    n_clusters: int = 3           # 聚簇数量（默认 3，跟当前场域数一致）
    max_iter: int = 100           # k-means 最大迭代
    random_state: int = 42        # 固定随机种子，保证可复现
    n_init: int = 10              # 初始化次数（选最佳结果）
    spherical: bool = True        # 球面 k-means（余弦距离）
    cluster_threshold: float = 0.3  # 低于此值视为"未分配"
    auto_k: bool = False          # 是否自动检测最佳 K（后期再开）
```

**为什么 n_clusters=3：** 跟现有场域数一致，用户后续可以在前端调整。

## Step 2：创建聚类引擎

**新文件：** `retrieval/cluster_engine.py`

### 2.1 核心逻辑

```python
class ClusterEngine:
    """
    职责：
      - 接收球体向量列表 → 跑 k-means → 输出簇中心 + 分配结果
      - 持久化/加载聚类状态
      - 新球体入库后触发
    
    不依赖 FieldDetector —— FieldDetector 反过来读这个引擎的结果。
    """
```

### 2.2 接口设计

```python
def fit_predict(self, vectors: np.ndarray) -> tuple:
    """全量聚类
    Args:
        vectors: shape (n, embed_dim), L2 归一化
    Returns:
        centroids: shape (k, embed_dim)
        labels: shape (n,) 每个球体所属簇 ID
        scores: shape (n,) 到所属簇中心的余弦距离
    """

def predict(self, vectors: np.ndarray) -> tuple:
    """对新向量预测簇归属（不需要重聚类时用）"""
    # 用已训练好的 centroids 做最近邻匹配

def save(self, path: str):
    """保存聚类状态（centroids + 参数 + 版本号）"""

def load(self, path: str) -> bool:
    """加载已保存的聚类状态"""
```

### 2.3 算法选择

用 **`sklearn.cluster.MiniBatchKMeans`**，不用 FAISS k-means：

| | MiniBatchKMeans | FAISS k-means |
|---|---|---|
| 增量学习 | ✅ `partial_fit()` | ❌ |
| 球面距离 | ✅ 手动归一化 | ✅ `spherical=True` |
| 随机种子 | ✅ 可复现 | ❌ |
| 依赖 | sklearn（可能没装） | 已有 |
| 速度(1500点) | ~20ms | ~20ms |

两者速度一样，MiniBatchKMeans 有 `partial_fit` 作为未来扩展选项。FAISS 的 k-means 更适合百万级量。

### 2.4 持久化格式

```python
{
    "version": 1,
    "n_clusters": 3,
    "n_samples": 1458,
    "centroids": [[...], [...], [...]],     # k × embed_dim
    "inertia": 12.34,                       # 簇内平方和（评估聚类质量）
    "created_at": "2026-07-13T15:00:00",
    "n_iter": 12,                           # 实际迭代次数
}
```

**存到：** `data/clusters/cluster_state.json`

## Step 3：FieldDetector 对接聚类

**文件：** `retrieval/field_detector.py`

### 3.1 改动

`FieldDetector` 当前的 `_centroids` 是从"标签均值"算的。改成从 `ClusterEngine` 读取：

```python
class FieldDetector:
    def sync_from_clusters(self, cluster_engine: ClusterEngine):
        """从聚类引擎同步 centroids"""
        centroids = cluster_engine.get_centroids()
        # centroids 没有"名字" → 用 cluster_0, cluster_1, ...
        # 或者保留旧 source_type 作为参考
        self._centroids = {
            f"簇{i}": centroid
            for i, centroid in enumerate(centroids)
        }
```

### 3.2 场域命名

聚类产生的簇没有名字，暂时叫 `簇0`、`簇1`、`簇2`。前端可以让用户重命名，然后持久化 `{cluster_id: display_name}` 映射。

启动时检测 `source_type` 和聚类的交叠——如果大部分"技术笔记"标签的球体落入了 `簇0`，可以把 `簇0` 自动建议命名为"技术笔记"。

**但这一步可以先不做——3.1 就够了。**

## Step 5：上传 + 聚类联动

**文件：** `api/main.py`

### 5.1 上传结束后（`upload_file` 函数末尾）

```python
# 上传结束后 → 触发聚类
# 条件：新增球体 > 0 且 auto_rebuild=True
if new_spheres > 0 and auto_rebuild:
    t_cluster = time.time()
    active_spheres = state.sphere_store.get_active()
    vectors = np.stack([
        state.faiss_store._vectors[
            state.registry.faiss_id(s.id)
        ]
        for s in active_spheres
    ], axis=0)
    
    centroids, labels, scores = state.cluster_engine.fit_predict(vectors)
    
    # 更新每个球体的 cluster_id + gravity_field
    for sphere, label, score in zip(active_spheres, labels, scores):
        sphere.cluster_id = int(label)
        sphere.gravity_field = {
            f"簇{j}": float(dist)
            for j, dist in enumerate(
                vectors[i] @ centroids.T  # 余弦距离
            )
        }
    
    # 同步到 field_detector（更新质心）
    state.field_detector.sync_from_clusters(state.cluster_engine)
    
    # 持久化
    state.cluster_engine.save()
    state.sphere_store.save()
    
    timings["clustering"] = time.time() - t_cluster
```

### 5.2 启动时加载

```python
# 在 startup 事件中
state.cluster_engine.load()  # 加载已有聚类状态
# 如果加载失败（首次运行），等第一次上传触发
```

## Step 6：检索聚焦适配

**文件：** `retrieval/retriever.py`

### 6.1 改动

当前 `field_focus` 是按 `source_type` 过滤。改成按 `cluster_id` 过滤：

```python
if field_focus:
    sphere = self.spheres.get(sid)
    if not sphere:
        continue
    # field_focus = "簇0" → 过滤出 cluster_id=0 的球体
    expected_id = int(field_focus.split("_")[1])  # "簇0" → 0
    if sphere.cluster_id != expected_id:
        continue
```

### 6.2 前端传参

前端传 `field_focus: "簇0"` 而不是 `field_focus: "技术笔记"`。这一步需要前后端对齐。

---

## Step 7：删除场景

当球体被软删除后：

1. 球体的 `active=False`
2. 不触发立即重聚类
3. 下次上传时，`get_active()` 不包含该球体 → 聚类自动排除它
4. 簇中心会自然漂移

**好处：** 不需要单独处理删除。上传即维护。
