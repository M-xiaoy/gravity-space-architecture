## 核心命题

**多样性维度是 softmax 完全丢弃的信息。**

Gravity Reranker 的核心思路：不修改 Transformer 内部，不搭建完整的两层架构——只做一件事——证明多样性信息是存在的、可提取的、对决策有价值的。

## 实验设计（2026-07-09 定稿）

### Gravity Reranker

```
输入问题
  ↓
LLM 多次采样（T 条不同的推理路径）
  ↓
对每条路径的每一步，记录：
  ├── 该步生成的 token
  ├── 该步的概率分布 (logits)
  └── 各注意力头对输入 token 的权重分布（如有）
  ↓
路径合并 → 每个候选答案的统计量：
  ├── 平均 log-prob（标准做法）
  └── 跨路径支持多样性（新做法）
  ↓
Gravity Score = avg_log_prob × (1 + α × diversity)
  ↓
按 Gravity Score 重排候选答案 → 输出
```

### 任务选择

**首选：矛盾文本推理**

给模型两段互相矛盾的文本，问一个需要交叉验证才能回答的问题。正确答案是「不确定」——因为两条证据路径对同一问题给出了矛盾的结论。

示例：
```
文本1：小明养了一只猫和一只狗。猫叫咪咪，狗叫旺财。
        每天早上咪咪会在窗台上等日出。

文本2：小明的宠物中只有狗会在早上叫醒他。
        猫通常要睡到中午才会活动。

问题：咪咪早上会在哪里？
  A. 窗台上  B. 窝里睡到中午  C. 不确定   → 正确答案：C
```

### 通过标准

满足以下任一条件即视为核心命题得到实验支持：
1. 准确率提升 ≥ 3%（绝对百分点）
2. 不确定性拒绝率提升
3. 错误自信率降低

## 实验协议 v0.1

> 2026-07-11 启动 | 目标：验证核心命题

### 假设

**H0（零假设）：** 多样性信息对决策质量无影响——Gravity Score 与标准平均 log-prob 在排序准确率上无显著差异。

**H1（备择假设）：** 多样性信息可提升决策质量——Gravity Score 在矛盾文本推理任务上的准确率 ≥ 标准方法 + 3%（绝对百分点）。

### 评价指标

| 指标 | 定义 | 目标 |
|------|------|------|
| 准确率（Accuracy） | 正确预测数 / 总样本数 | ≥ +3% 绝对百分点 |
| 不确定性拒绝率 | 预测为「不确定」且正确 | 提升 |
| 错误自信率 | 自信地给出错误答案 | 降低 |

## 实验结果 v0.2

**Run ID:** run_009 | **日期:** 2026-07-11
**模型:** Qwen/Qwen2.5-0.5B（eager attention）
**方法:** 单次前向 + 跨头注意力多样性分析

### 结果摘要

| 指标 | 值 |
|------|----|
| 选择题准确率 | 0.0% (0/6) |
| 开放题准确率 | 68.8% (11/16) |
| 不确定题准确率 | 25.0% (1/4) |
| 平均多样性比率 | 0.091 |
| token 选择改变 | 0/2102 |

### 核心发现

**发现 1：注意力头几乎不分散。** 336 个 head 中，约 90% 集中在 BOS token（位置 0）和少数几个 token 上。平均多样性比率仅 0.091。

**发现 2：0.5B 模型 attention heads 无专门化。** 各 head 没有形成证据追踪分工——不同 head 在矛盾文本之间分裂的假设不成立。

**发现 3：token 选择零改变。** 调整偏差 `α × (diversity - 0.5) = 0.3 × (0.091 - 0.5) = -0.123`，不足以翻转任何 token 排名。

### 诊断结论

**核心命题「多头注意力中隐藏着证据多样性信息」：** ❌ 在 0.5B 上未观察到。

根本障碍：0.5B 的 336 个 attention heads 没有足够的模型容量来实现专业化分工。7B+ 模型的 heads 才可能分化。
