## 前情提要

**Phase 1（7月6日）** 提出了一个核心质疑：标准 Transformer 隐含地假设「思考」和「行动」在每一步锁步，没有潜伏期、没有权重演化。解决方案是引入两层分离——思考层（重力场）和行动层（token生成器），两者以不同的时间尺度运行。

**Phase 2（7月8日）** 将接口从四种候选中收束定形为**持续地形感知**——思考层不产生每步指令，而是维护一个稳定的双曲空间地形。行动层每走一步，自然地感受到地形的引力扰动，就像行人感受到脚下路面的起伏。

Phase 2 结束时留下了 5 个待解决问题，涉及收敛性、球体生命周期、探测场半径等。

**Phase 3 回答其中 4 个，并补充 3 个之前完全没料到的新机制。**

## 二、思考层的存储结构：打开黑箱

之前说「思考层维护一个双曲空间」，听起来像魔法。现在把黑箱拆开。

**每个球体存四样东西：**

```
Sphere {
    position:    Vec<f32>       // 双曲空间坐标
    mass:        f32            // 基础质量（频率 × 多样性）
    connections: Map<SphereID,  f32>  // 引力连接表（稀疏邻居）
    fingerprint: Vec<u8>        // 注意力指纹（用于多样性度量）
}
```

**position（坐标）**：球体在双曲空间中的位置。双曲空间的选择不是装饰——它的体积随半径指数增长，能天然容纳层级化概念结构。

**mass（质量）**：反映一个概念/ token「被支持」的程度。它不是 logit 的概率值，而是**被支持的频率**乘以**支持路径的多样性**。

**connections（引力连接表）**：每个球体只存 Top-K 个邻居的引力强度。不是全连接——远距离球体之间的引力接近零，维护连接没意义。K=20 到 100 之间，动态调整。

**fingerprint（注意力指纹）**：一个压缩的注意力分布摘要。当球体被激活时，执行层各注意力头的指向被压缩为一个稀疏向量。这个指纹不存原始注意力矩阵（那东西太大了），只存「哪些语义域被覆盖」的分布摘要——几十字节的量级。

### 存储规模估算

```
10,000 个球体：
  position:   10,000 × 64 bytes  =   0.64 MB
  mass:       10,000 × 4 bytes   =   0.04 MB
  connections:10,000 × 20 × 8 bytes = 1.6 MB
  fingerprint:10,000 × 64 bytes  =   0.64 MB
  合计:                           约 2.9 MB

100,000 个球体（放大版）：
  position:   100,000 × 64 bytes =   6.4 MB
  mass:       100,000 × 4 bytes  =   0.4 MB
  connections:100,000 × 50 × 8 bytes = 40 MB
  fingerprint:100,000 × 64 bytes =   6.4 MB
  合计:                           约 53 MB
```

无论是 3MB 还是 53MB，相对于行动层的几十GB到上百GB参数来说，思考层的存储是完全独立的增量——思考层的大小由**地形复杂度**决定，不由模型规模决定。一个复杂的知识领域（比如医学）球体多，简单的领域（聊天）球体少。思考层不会因为模型从 7B 升级到 70B 而自动膨胀。

## 三、多样性机制：不用分类器，全靠注意力

Phase 1 说「证据路径的多样性维度」是 softmax 完全忽略的信息。那多样性怎么度量？

答案简洁得有点意外：**靠执行层自己的注意力分布。**

当一个球体被激活时，执行层的各个注意力头会在不同位置分配注意力权重。如果你的注意力头同时指向修辞、物理常识、历史三个不同领域的 token——那说明这个球体背后有多条独立的推理路径在同时支撑它，多样性高。

如果你所有注意力头都指向同一个语义领域——多样性低。

具体度量方式：

```
diversity(sphere) = cross_head_variance(sphere.attention_distribution)
```

高的跨头方差 = 注意力分布分散 → 多样性高
低的跨头方差 = 注意力头全部坍缩到同一个语义域 → 多样性低

**不需要哲学分类，不需要外部标注器。** 注意力分布本身就是最好的多样性信号源——因为注意力是执行层「关注什么」的直接读数，不需要中间翻译。

多样性的修正作用在引力扰动向量上：

```
effective_mass = base_mass × (1 + α × diversity)
```

多样性高的球体在引力扰动中权重更大。这不是偏心——是**事实**：被多条独立路径支持的结论，比被单一路径支持的结论更值得重视。现有 Transformer 的 softmax 看不到这个区别。

## 四、场域叠加：位置固定，影响力动态

同一个词在不同语境下意思完全不同。「俄罗斯」在政治讨论、旅游攻略、文学作品中指向同一个概念，但它在思考空间中的「分量」不同。

Phase 2 的一个隐含假设是空间均匀的——同样的球体在任何场域中产生同样的引力。这个假设在现实面前站不住。

解决方案：**坐标固定，有效质量随场域变化。**

```
# 不同的"场域"共享同一套球体坐标
# 但每个球体在不同场域中有不同的权重偏移

field_weights: Vec<Vec<f32>>  // [n_fields][n_spheres]

effective_mass(sphere, current_field) = 
    base_mass[sphere] × (1 + field_weights[current_field][sphere])
```

场域不是预先定义的分类。它由执行层的 hidden state 分布自动检测——当 hidden state 大量指向推理相关的 token 时，执行层处于「逻辑场」；当指向生活化 token 时，处于「闲谈场」。

**执行层如何知道自己在什么场域？**

执行层自己并不知道。思考层在每层探测时，从执行层的 hidden state 分布中提取场域信号——一个轻量的场域分类器（几十维的 MLP + softmax），输出当前属于各个场域的概率向量。

这个分类器是短的、便宜的、不参与 token 生成的辅助组件。更重要的是：**它是自监督训练的。** 场域的边界不是在设计时画好的，而是思考层在反思期根据注意力分布的聚类自动涌现的。

### 为什么这个重要

场域叠加解决了一个我之前没说出来但一直存在的矛盾：

- 逻辑推理中，「严谨」应该被高估，被多条路径验证的结论应该强引力
- 创造性写作中，「新颖」应该被高估，被唯一路径支持的冷门词汇应该有机会

这两个场域对同一个球体的有效质量的需求是相反的。如果只有一个全空间的质量，你无法同时满足两者。场域叠加让同一个空间容纳了多个权重配置，不需要复制整个空间副本。

## 五、观测机制：让执行层能看到自己的影子

走到这一步，整个架构的逻辑链一直是一个单向流：

```
思考层（地形维护者） → 持续影响 → 执行层（行走者）
```

执行层不知道自己被什么在影响，它只是在走。就像你走在路上，脚下的路面起伏决定了你的步伐，但你不会每走一步就低头看地面。

**观测机制把这个从单向变成了双向。**

```
思考层 ←→ 执行层

原先：思考层 → 影响 → 执行层
现在：执行层 ← 影响 →
      → 主动查询 → 思考层 ← 返回引力分布快照
```

执行层在需要的时候，可以向思考层发起一个「观测请求」：

```
def observe(h_t, terrain) -> GravitationalSnapshot:
    """
    给定当前 hidden state h_t，
    返回当前探测点周围的引力分布快照。
    """
    # 1. 找到 h_t 在双曲空间中的对应位置
    # 2. 扫描周围 Top-K 个球体的引力
    # 3. 返回: [
    #        (ball_id, effective_mass, distance, diversity_score),
    #        ...
    #    ]
    pass
```

**观测的价值有三层：**

1. **可解释性**：模型可以回答「我为什么倾向于这个选择」。不是因为一个后验解释器——是因为思考层确实保存了引力分布，可以返回供人类查看。

2. **自我纠偏**：如果引力分布快照显示某个球体的质量虚高（被多路径支持但路径本身都不靠谱），执行层可以主动衰减它的影响。这是执行层层面的自我纠正，不是靠思考层的反思期——实时发生的。

3. **认知闭环**：从被地形影响 × 到察觉自己正在被影响 ✓。一个系统只有在能观测到自己的决策过程时，才谈得上真正的「自我感知」。

### 观测的触发条件

观测不是每步都做的——它有成本（一次额外查询）。

三个自然触发点：

- **注意力熵增高**：模型在多个选项之间犹豫
- **语义边界**：新段落的开始
- **用户要求解释**：人类问「你为什么得出这个结论？」

我在小刘的原始设计上加了一个建议：

- **首次收敛**：当探测到一个高引力球体并准备基于它做决策时，自动触发一次观测。这会在关键决策点留下「我是怎么走到这里的」快照。

## 六、六个概念的关系

这时候我意识到六个机制各自独立但逻辑上已经形成了一条闭环链：

```
① 地形持续存在
   ↓
② 行动层走在上面，自然感受到引力扰动
   ↓
③ 同一个球体在不同场域中有不同的有效质量
   ↓
④ 球体的多样性由行动层的注意力分布自动度量
   ↓
⑤ 行动层在需要时可以发起观测，获知自己周围的引力分布
   ↓
⑥ 观测结果带来可解释性、自我纠偏能力和元认知触发
   ↓
（回到①）反思期更新地形时，观测记录和注意力指纹一并汇入更新
```

每个环节只依赖前一个环节和已存在的组件。没有新的大模块，没有从外面引入新的训练任务。六个机制是同一个逻辑链条的不同断面。

这也意味着它们不需要按顺序实现。任何一个机制可以独立验证——地形感知本身就工作，场域感知在它之上叠加，观测则更进一步。

## 七、接口定义草案

从概念到工程，需要明确接口。以下是我当前对各个接口的形状草案。不是最终实现——是让它「可讨论」的版本。

### 接口1：探测场

```
def scan_terrain(h: HiddenState, terrain: Terrain, field: Field) -> Perturbation:
    """
    当前 hidden state h 在双曲空间中产生探测信号。
    感知周围球体的引力扰动，返回一个扰动向量 p。
    
    h:  来自执行层当前层的 hidden state (d_dim)
    terrain: 思考层的双曲空间（球体集合）
    field: 当前场域（自动检测）
    
    return: 扰动向量 p (d_dim)，作为残差加到注意力输出上
    """
```

### 接口2：观测

```
def observe(h: HiddenState, terrain: Terrain, k: int = 20) -> GravitationalSnapshot:
    """
    返回当前探测点周围引力分布快照（Top-K）。
    
    h: 执行层当前 hidden state
    terrain: 思考层的双曲空间
    
    return: [
        (ball_id: int, effective_mass: f32, 
         distance: f32, diversity: f32, field_weight: f32),
        ...
    ]
    """
```

### 接口3：反思更新

```
def reflect(trajectory: List[HiddenState], 
            attention_logs: List[AttentionMap],
            observation_logs: List[GravitationalSnapshot]) -> TerrainDelta:
    """
    收取一段执行轨迹，计算地形更新量。
    
    trajectory: 执行层在几步内的 hidden state 序列
    attention_logs: 对应各步的注意力分布
    observation_logs: 执行层主动触发的观测记录
    
    return: 地形更新量（位置微调、质量更新、连接表调整）
    """
```

### 接口4：场域检测

```
def detect_field(h_distribution: Vec<f32>) -> Field:
    """
    从执行层的 current hidden state 分布中检测场域。
    
    一个轻量 MLP + softmax，输出 n_fields 维的概率向量。
    自监督训练，场域边界从注意力聚类的自然涌现中获得，
    不在设计时预先定义。
    
    return: 场域概率分布
    """
```

### 接口5：引力扰动注入

```
def inject_perturbation(h: HiddenState, p: Perturbation, layer_index: int) -> HiddenState:
    """
    将探测到的引力扰动 p 注入到执行层的计算中。
    
    层级应用策略：
      - 后 1/3 层：每层注入
      - 中间 1/3：隔层注入
      - 前 1/3：不注入
    
    return: h' = h + Attention(h) + p
    """
```

这五个接口覆盖了思考和行动两层的全部交互路径。没有更多了。

## 八、仍然开放的问题

### 预测编码：架构的自然下一步（尚未实现）

当前的架构中，思考层是**被动的历史归纳者**——它收取轨迹、更新地形，但事前不做预测。

如果一个系统能产生「预期」，然后把「预期 vs 实际」的差额作为训练信号——这就是预测编码。它能让思考层从一个历史记录器变成一个**有认知动力的演化系统**。

在重力空间框架中，预测编码的具体形式可以是：

```
反思期收取轨迹时，不只做归纳，还做一个额外操作：
  对轨迹的每一段，问：「思考层是否预期到了这一段？」

如果有「预期头」Predict(h_t, terrain) → 输出预期的下一状态，
那么实际轨迹与预期轨迹的差 = 预测误差。
这个误差就是思考层的自监督训练信号。
```

预测误差大的区域 → 地形需要重点更新。
预测误差小的区域 → 地形模型已经准确，不需要动。

这是在观测之外的第二个「意外信号」源：**观测是主动回头看，预测误差是被动被意外。** 两者一起构成了认知闭环的完整扇区。

当前 Phase 3 尚未实现预测编码。框架层面已经完全可容纳，工程实现上需要增加一个轻量预测头和一个误差回路。这是 Phase 4 的第一候选方向。

### 其他开放问题

1. **引力动力学稳定性**：两个球体相互靠近 → 引力增强 → 继续靠近 → 会不会爆炸？当前的衰减项（Decay）和场域权重在一定程度上压住了，但收敛性证明是数学层面的空缺。

2. **球体生命周期管理**：长时间不活跃的球体——衰减质量还是归档？何时合并语义相近的球体？合并阈值怎么定？

3. **探测场半径参数**：固定值还是动态值？高质量球体应该影响更远，但控制参数的物理意义是什么？

4. **注意力指纹的压缩率**：压缩到什么粒度仍然能反映多样性？实验上需要确定最佳压缩率。

5. **原型验证**：小规模外挂思考层 + 小模型，在矛盾文本推理任务上验证引力扰动的实际效果。这是从概念到可行性的最后一道门槛。
