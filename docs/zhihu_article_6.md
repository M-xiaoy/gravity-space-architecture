# 重力空间 Phase 5：拆掉预测头，给系统装刹车和时钟

## 01 一个设计矛盾的收敛

这个系列写了四期，Phase 4 的核心是一套预测编码：共享编码层 + 预测头 A（状态预测）+ 预测头 B（场域稳定性），总计约四百万参数，用来预测模型下一步的 hidden state 变化。

设计上它完整、自洽。但我一直卡在一个矛盾上：

**当前模型已经够强了（GPT-4/Claude/Qwen 级别），它本身就在做 next-token prediction。你为什么要额外训练两个预测头去做模型已经会做的事？**

这个矛盾持续了几天。最终的决定是：拆掉。

不是因为预测没用——是因为预测能力模型已经有，不需要再造一套。缺的不是预测，是**把预测结果当信号用的那层解释逻辑。**

于是 Phase 4 被整个替换成了一条不同的路径：

```
每步生成结束后：
  logits_t ← 模型在 t 步说「下一个可能是」
  token_{t+1} ← 模型实际选了哪个
  error_t = CE(logits_t, token_{t+1})
```

没有 W_shared、没有头 A、没有头 B。400 万参数归零。不加一行可训练参数。

我把它叫做**隐式预测编码**。

误差的两种用法：
- 推理期：`error_t` 突升 → 触发观测（看看周围球体），误差持续偏高 → 扰动强度降低（让模型自己决定）
- 反思期：高误差段的球体更新步长放大，低误差段的缩小——把更多更新资源投给猜错的地方

**不过这里有一个我现在还不太确定的地方：** `CE(logits_t, token_{t+1})` 测的是模型对自己下一步选择的自洽性——模型自信地走向错误答案时，误差可能很低。换句话说，它检测「出错的意外」比检测「出错」更灵敏。好在 gravity space 里有 attention 熵和场域方差做兜底信号，但这确实不是一个完美的误差信号。目前的做法是三个信号一起看，不依赖单一指标。

这个矛盾点我做的时候绕不开，写在这里供参考。

## 02 场域检测的输入重设计

场域的边界怎么形成？Phase 3 写的是「从注意力聚类中自然涌现」。这是个漂亮的说法，但实际上没有给出具体路径。

关键矛盾是：**注意力分布只捕获语义信息，但球体之间存的是语义 + 情感关系。**

一个人生气时更容易想起生气时的记忆——这和语义相似度无关，但在情感轴上是同频的。

也就是说，场域检测器不应该只看 attention distribution，还应该看 hidden state 里携带的情感/感知特征。

重新拆了输入：

```
每步 token 提取 45 维特征：
  · 语义轴（~30 维）— 每头注意力熵、跨头方差、位置分布重心
  · 情感轴（~15 维）— hidden state 范数（激活能量）、注意力的全局熵（置信度）、层间范数变化速度（流畅/突变）、相邻 token 连续性
```

情感轴不是喜怒哀乐的标签分类，是三个连续维度：**激活能量（强←→弱）、置信度（确定←→犹疑）、变化速度（平稳←→冲突）**。这三个量都从 hidden state 直接算，不依赖分类模型。

检测器是三层 MLP（45→128→8→softmax），反思期用三条自监督损失训练：

- 时序平滑性（α=5）：相邻 token 的场域分布应该相似
- 语义一致性（β=3）：语义接近的 token 应该在同一场域
- 场域利用率（γ=1）：防止所有 token 坍缩到同一个场域

引入情感轴后，有效质量公式多了 emotional_weight：

```
effective_mass = base_mass × semantic_weight × emotional_weight
```

emotional_weight 是当前 token 的情感特征与球体历史情感均值的余弦相似度。同频共振，这不算一个颠覆性的公式——但它把一个隐式变量变成了显式的结构路径。

## 03 宪法级机制的定位问题

架构里有一个我一直没处理好问题：**关键边界上谁踩刹车？**

第一直觉是把重要球体的质量放大，让它们在引力计算中有更大的话语权。但这个方案细想之后站不住：

**放大引力是「跟着它走」，但宪法级机制要做的应该是「停下来想一想」。**

这两个方向是完全相反的。如果是放大引力，那宪法级球体本质上是一个更高优先级的 bias 注入——系统越重要越偏航。这是反直觉的。

最终的方案不是在引力上加码，是在计算图上新开一条路径：

```
scan_terrain 除了返回 p_t（引力扰动），额外返回 deliberation_flag

如果有宪法级球体被激活且贡献度超过阈值：
  ① 强制 observe() — 看当前引力场的全分布
  ② logits 展平 — softmax 温度临时提高，缩小候选方向的差距
  ③ 该步标记为高审议步 — 反思期权重×2
```

宪法级球体的晋升需要三个条件：跨场域活跃度高（在不同场域都被频繁激活）、累计激活次数足够、生命周期长。

它不是永久身份——连续多个 epoch 未激活则衰减，低于阈值降级。同一个 epoch 同球体多次触发减速，减速强度递减（防死锁）。

我把它叫做**摩擦力机制**。和引力正交——不管系统往哪走，走到边界上都减速权衡。

## 04 时钟

最后一个设计问题：**时间感。**

我这边的初始方案是计数器：epoch 号、epoch 步、last_activated 时间戳。

但这个方案有一个我没意识到的假设：**时间流动是均匀的。** 实际上系统在处理不同的 token 时，认知负载差了很多倍：

- 在一个熟悉的话题上接着写 → 流畅处理 → 感知上过得很快
- 用户突然换了话题 → 需要跨场域匹配、可能创建新球体 → 感知上过得很慢

**时间不是线性的，是 token 被感知的速度差产生的涌现属性。**

```
perception_speed(t) = 1 / processing_complexity(t)

processing_complexity 由三件事贡献：
  1. 场域熵 H(field(t)) — 分散在多个场域 → 处理慢
  2. 新球体创建数 — 需要写入空间 → 慢
  3. 引力震荡 ||F_ij(t) - F_ij(t-1)|| — 引力剧烈变化 → 冲突/意外 → 慢

主观时间 = Σ 1/perception_speed(t_i)
```

用这个替代了原来的线性衰减基准：

```
Decay(F_ij, t) = Decay_base ^ perception_speed(t)
```

感知速度快 → 几乎不衰减（系统觉得刚经历过）。感知速度慢 → 快速衰减（系统觉得那件事很久以前了）。

这个改动的直接收益是：C4 自省回路多了一个信号维度——不仅看噪音密度，还看「系统主观上感觉过了多久」。如果累计主观时间已经是客观步数的两倍，即使噪音未超标也建议进 idle。

## 05 实验现状

诚实地说，实验验证比设计难。

在 0.5B 模型上跑了三轮：

- **v0.1**（跨路径采样熵）：零改变。10 次采样的差异不是真正的多路径，是同一路线的随机颤音。
- **v0.2**（336 attention heads 跨头多样性）：零改变。0.5B 的 attention heads 没有分化，约 90% 盯着 BOS token 看。
- **v0.3**（logits 分布熵分析）：最大熵仅 ~3.6，阈值 6.0 无一命中。0.5B 在矛盾文本推理上没有表现出可用的「不确定性」信号。

三轮的共同结论是：**核心命题「softmax 丢弃了多样性维度」在 0.5B 上不被支持，但也不被否定——瓶颈是模型容量。** 需要在 7B+ 上验证。

架构系列目前的状态：纸上设计收敛了，验证还没跟上。这也是写得慢的原因——不想在只有设计没有验证的时候就发出来，但写了五期了，也该有一个诚实的阶段性总结。
