> 第二阶段（2026-07-08）：接口从 hidden state 层级收束到 logits 层级，再收束到持续地形感知
> 
> 本文同步发布于知乎。

---

## 零、第一阶段回顾

重力空间架构的核心主张：

**标准 Transformer 隐含一个假设：「思考」和「行动」在每一步锁步。** 每生成一个 token，模型重新计算一次注意力——这是它的「思考」。但这个思考没有潜伏期，没有权重演化，每步都从头开始。

我们提出：模型推理需要两个时间尺度不同的子系统——

- **行动层**：标准 Transformer，步频 = token 级，负责任务执行
- **思考层**：双曲空间中的球体引力模型，步频 = 语义回合级，负责多源证据积累

两层的交互方式，上一版探讨了四种方案（增益控制/势场调制/慢上下文/表示剪切），但卡住了——因为这四种都在表征空间层面做动作，要求思考层理解行动层的内部结构，对思考层来说过多且不必要。

---

## 一、第一次收束：可加偏置向量

第一步思考：**思考层不需要知道 token 的内部表征，它只需要在输出空间告诉行动层「哪个方向有更多的证据」。**

于是接口被收束为：思考层的产出是一个偏置向量 `b ∈ R^|V|`，与词表同维，加到行动层的 logits 上。

```
final_logits = lm_logits + b
```

这个收束很干净：零侵入、自然稀疏、可端到端训练。

但它有一个严重的固有缺陷。

---

## 二、可加向量的问题：一次即弃

思考层产出一个 `b`，加到这一步 `lm_logits` 上，做一次采样——然后 `b` 就消失了。下一步行动层推理时，模型用的是自己的上一步输出作为输入，`b` 不在了。

这意味着：

1. **影响是一次性的。** 它只修正当前 token 的选择，不修正后续推理的走向。叫做「影响」不如叫做「拨正」。

2. **本质上等同于预设路径。** 模型已经算完了整条前向传播，思考层在最后拉一把。如果模型前几步就走偏了，这一步的偏置拉不回来，因为它只作用于最终 logits。

这不是我们想要的「永远影响」。我们希望的是：

> 思考层的影响像地形——执行层在上面行走，每一步都踩在地形上，而不是只在跳起来的那一下被推一把。

---

## 三、第二次收束：从单次偏置到持续地形感知

新的设计：

**思考层不产生每步的指令。它只维护一个稳定的双曲空间——一个地形。**

行动层推理时，每一层（或每 N 层）的当前状态在双曲空间中产生探测场，感知周围球体的引力扰动，这个扰动的结果注入到那一层的计算中：

```
每层 forward 时：
    当前 hidden state h → 探测场扫描双曲空间 → 引力扰动向量 p
    h' = h + Attention(h) + p
```

这里的 `p` 和 `b` 有本质区别：

- `b` 是思考层产出的固定指令，每步产生一次，用完消失
- `p` 是行动层自己在双曲空间中**感知到的**——空间没变，但每层的 h 不同，在空间中的位置不同，感知到的扰动不同

**地形（双曲空间）在推理过程中是稳定的**——思考层只在回合之间更新它。所以 `p` 不是思考层的「输出」，而是行动层**自主的感知结果**。

这意味着：
- 思考层不告诉行动层「你该怎么做」
- 思考层只保证：地形始终在那里，只要你踩上去，你就会感受到证据的引力
- 行动层走哪条路，取决于它自己——但每一步都自然地受到了证据分布的影响

「永远影响」不是靠每步注入一个新信息实现的，是靠**地形不随时间衰减**实现的。

## 四、成本控制：不便宜的接口

方向B有一个现实代价：每层多一次双曲空间查询。

但这个问题可以分拆解决：

### 4.1 权重预分配

行动开始前，思考层对双曲空间的不同区域分配权重配比。不是所有球体在所有语境下都有相同的感知优先级——一个关于编程的推理，不需要感知「食堂晚餐」对应的球体群。权重预分配可以让探测场的扫描范围缩窄一个数量级。

### 4.2 动态查询深度

不是每层都要查。浅层处理的是局部语义，深层处理的是全局推理——引力扰动对深层的价值远大于对浅层的价值。可以在深层（后 1/3 的层）每层都查，中层（中间 1/3）隔层查，浅层（前 1/3）不查。

### 4.3 间隔查询

更激进的做法：固定间隔 N 步查一次，中间步重用上一次的 `p`。因为地形是稳定的（思考层只在回合间更新），间隔查询丢失的不是地形的变化，只是行动层位置变化导致的感知差异。如果 N 不大（比如每 3 层），这个差异可以忽略。

这三种策略可以叠加组合。最终成本可控——在不牺牲核心效果的前提下。

## 五、时序结构：仍然是交替模式

接口变成持续地形感知后，时序不变：

```
行动层运行期 ──────────────────
    思考层静默，空间不变
    执行层每 N 层感知引力扰动
                                ↓ 语义边界 / 注意力熵高 / 输出完成
思考层高算力期 ────────────────
    收取行动层轨迹 → 归纳 → 更新球体质量和位置
                                ↓ 更新完成
下一个行动层运行期 ─────────────
    思考层回到静默，但地形已经变了
```

思考层产生的不是偏置向量，是地形的更新。更新频率不耦合于 token 步频。

---

## 六、双曲空间的具体设计（重申）

**球体 = token / 概念。** 每个球体有位置和质量。

**质量 = 频率 × 多样性。**
- 频率：被调用了多少次
- 多样性：被多少条独立的推理路径支持

多样性是核心——它在 softmax 中被完全忽略。softmax 只看单步的数值大小，不看「有多少条不同的证据路径同时指向同一个选项」。多样性维度是重力空间架构对概率推理的根本补充。

**探测场：** 当前 hidden state h 在空间中产生一个探测信号，感知周围球体的引力扰动。引力扰动 = 质量 / 距离^α。不涉及注意力计算，不涉及可学习参数——纯几何操作。

**探测场 → 表征扰动 p：** 周围所有球体的引力扰动叠加，经一个线性投影映射到 h 的同维空间，作为残差加到注意力输出上。

## 七、反思机制

思考层在回合间收取行动层的轨迹，用来更新双曲空间。

轨迹包括：
- 实际被采样的 token 序列
- 注意力分布（哪些 token 之间有高权重）
- 路径中断点（注意力分布的突然变化）

归纳方式：
- **频率更新**：实际被采样的 token 频率 +1
- **多样性更新**：跨注意力头的注意力权重冗余度 → 如果多个注意力头同时关注同一组 token，说明这些 token 被多条路径支持 → 对应的球体多样性增加
- **位置微调**：两条轨迹中经常同时激活的球体相互靠近（引力牵引）
- **新球体创建**：一个概念反复出现但空间中没有对应球体 → 创建

**收敛信号：** 思考层的「预期引力分布」与行动层实际行为之间的差距在跨回合缩小。这个差距是自监督的——不需要外部标注。

## 九、仍待解决的问题

1. **收敛性证明：** 引力扰动叠加的动力学是否稳定？会不会出现两个球体相互靠近→引力增强→继续靠近→爆炸？
2. **球体生命周期：** 长期不活跃的球体衰减还是归档？球体是否合并？
3. **探测场半径：** 固定值还是动态值？高质量球体应该影响更远，但控制参数是什么？
4. **间隔查询的最优 N：** 层数越深价值越大——但浅层完全放弃是否有副作用？
5. **原型验证：** 搭一个小规模原型（小模型 + 外挂思考层），在一个需要多源证据积累的任务（比如矛盾文本推理）上验证偏置效果。

这些是下一步的事。架构走到这里，从哲学比喻到可讨论的工程方案，已经跨过了本质的门槛。
