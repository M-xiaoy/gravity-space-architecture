## 二、核心概念

### 球体（Sphere）
最小的知识单元。一个文本切片 + 向量 + 元数据。

```python
Sphere(
    id: str,              # SHA256(text+source)[:12]
    text: str,            # 原文
    source_file: str,     # 源文件名
    source_type: str,     # 旧场域标签（保留用于兼容/命名参考）
    cluster_id: int,      # 所属聚类簇（-1=未分配）
    gravity_field: dict,  # {簇名: 引力值} 到各质心的距离
    mass: float,          # 基础质量
    active: bool,         # 软删除
)
```

### 场域与聚簇
- **场域不再由用户定义**，由 K-means 聚类自动发现
- 聚类中心 = 场域质心
- 球体归到最近的质心 → 形成聚簇
- 新增球体 → 触发全量重聚类 → 质心漂移 + 组重分配
- 删除球体 → 下次聚类自动排除

### 重力场（Gravity Field）
每个球体到所有场域质心的余弦距离。
- 由系统自动维护（聚类 → 质心变化 → 全量重算）
- 用于检索时的场域亲和度展示
- 不用于聚焦过滤（聚焦走 `cluster_id`）

## 三、完整检索流程

```
用户输入 "什么是跨来源优先？"

 1. embedder.embed_query() → query_vector (768d)
 2. FAISS search(fetch_k=100) → 100 个候选 ID
 3. Registry → Sphere IDs
 4. ── 场域聚焦处理 ──
    如果 field_focus="簇1" → 只保留 cluster_id=1 球体
    如果 exclude_ids → 排除已返回的球体
 5. FieldDetector.detect(query_vector) → 场域亲和度
 6. DiversitySorter.sort(MMR+多样性+场域加权) → Top-5
 7. 返回结果 + 聚合场域亲和一个 + 会话 ID
```

### 聚焦模式

| 模式 | 触发方式 | 行为 |
|------|---------|------|
| 无聚焦 | 不传 field_focus | 全库检索，多样性排序 |
| 聚焦 | `field_focus: "簇0"` | 只在该簇内检索 |
| 持续聚焦 | session 中的 field_focus | 自动沿用，直到 reset |
| 退出聚焦 | `reset_focus: true` | 清空 session 的聚焦状态 |

## 四、聚类维护流程

```
上传文件
  → 解析 → 切片 → 嵌入 → 入库
  → 触发全量 K-means 聚类
    → 收集所有活跃球体向量
    → KMeans(n_clusters=3).fit_predict(vectors)
    → 更新每个球体的 cluster_id
    → 重建 gravity_field（到新质心的距离）
    → 同步 field_detector 质心
    → 持久化聚类状态 + 球体库
```

- 启动时：如果有保存的聚类状态，用 `predict()` 恢复球体归属
- 启动时：如果没有聚类状态但有球体，触发首次聚类
- 删除：球体 `active=False` → 下次聚类自动排除
- 不依赖定时任务，不上传就不聚类
