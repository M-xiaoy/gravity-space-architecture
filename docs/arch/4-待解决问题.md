## 一、引力动力学稳定性

**编号：** GS-01
**来源：** Phase 3 遗留（2026-07-09）
**分类：** ✅ 已解决（见 ADR-011）

**问题：** 两个球体在双曲空间中互相靠近 → 引力增强 → 进一步靠近 → 会不会引力爆炸？

**解决方案（2026-07-11）：** 三阶稳定性体系——不依赖单一机制，不要求数学收敛证明。

| 层 | 机制 | 防什么 |
|----|------|--------|
| 1 | 场域重叠 Ω_ij 衰减跨域引力 | 跨域球体异常合并 |
| 2 | Decay(F(t)) 时间衰减 | 同域震荡放大 |
| 3 | m_i ≤ M_max, F_ij ≤ F_max（硬上限） | 单球体失控 |

**决策日后的状态：** 设计问题已关闭，工程参数（M_max/F_max阈值）留待实验。

**详见：** ADR-011（3-决策记录.md）、C2.8（2-组件全景.md）

## 七、噪音度量方法

**编号：** GS-07
**来源：** Phase 5 新增（2026-07-11）
**分类：** 🟡 公式已定，阈值待实验

**问题：** 噪音 = 重复出现的旧信息抑制新内容。怎么量化？

### 公式推导

**核心思想：** 噪音不是重复率，是引力场中的势能差。

**Token 级噪音贡献：**
```
n(t) = ||p_t||² / ||h_t||²

其中：
  p_t = scan_terrain(h_t)  — 新 token 受到的引力扰动（来自所有已存在的球体）
  h_t = token 的 hidden state
  分子：旧球体对当前 token 的总引力强度
  分母：当前 token 自身的信号强度
  比值：新 token 的「站稳难度」
```

当 `n(t) ≈ 0`：token 几乎不受现有引力影响 → 全新感知，毫无站稳压力 → 低噪音
当 `n(t) >> 0`：token 被旧球体强烈拉扯 → 旧信息在压制这条新信号的落地 → 高噪音

**注意：** `n(t) ≈ 0` 也可能意味着 token 完全落在了现有球体的引力盲区——这同样不是噪音，而是「未被触及的空白区域」，是好事。

**思考层整体噪音水平（自省回路使用的值）：**
```
N(t) = EMA(n(t), λ=0.95)

其中：
  EMA  = 指数移动平均
  λ    = 0.95 → 约 20 个 token 的有效窗口
  t    = epoch_step 内的当前步数
```

EMA 的物理意义：噪音不是单次事件，是**地形的持续状态**。一个 token 被拉不能说明什么，连续 20 个 token 都被拉——说明地形饱和了。

**噪音速度（二阶量）：**
```
ΔN = N(t) - N(t - window)
```

- ΔN 持续高 → 地形快速饱和 → 信号：快要触发 idle 了
- ΔN 低但 N 高 → 地形已饱和稳定 → 信号：已经在噪音中工作了
- ΔN 负 → 地形正在恢复（罕见，除非用户手动切换话题或 session 刷新）

### 阈值建议（初期实验值）

```
N < 0.1  → 清醒模式，P_t 全量输出
0.1 ≤ N < 0.3 → 正常范围内（周期中段的自然饱和）
0.3 ≤ N < 0.6 → 观察者模式：P_t 衰减到 0.5×
N ≥ 0.6  → 触发 idle（地形需要清理）
```

这些阈值是工程初始值，需要实验校准。

### 时间尺度

- **EMA 窗口（λ=0.95 → ~20 tokens）：** 决定了「多短的重复算噪音」。20 个 token 内的重复 → 被 EMA 捕获。隔了 500 个 token 的同一概念再次出现 → EMA 已衰减，不算噪音，算重新激活。
- **跨会话：** epoch 重置后 N 清零。跨会话的旧信息重现 → 如果在新 epoch 的前几个 token 就重现旧球体，它的引力仍在（球体未消失），但 N 从零开始爬升——噪音累积需要时间。

### 未定项

- ❓ 阈值需要实验校准——这些初始值来自架构直觉，没有数据支撑
- ❓ λ 的合理性——20 token 窗口是否足够？可能在实际对话中需要更大窗口
- ❓ 噪音速度 ΔN 是否纳入自省回路决策？还是只用 N？
- ❓ idle 后 N 重置为 0——但旧球体只是被清理/归档了，不是消失了。清理后的残存引力怎么处理？

### 在架构中的位置

- `n(t)` 在感知过程（C2.1.5）中实时计算——每次 scan_terrain 都能拿到 p_t
- `N(t)` 和 `ΔN` 在 C4 自省回路中维护——作为 `self_check()` 的核心输入
- 阈值 `T_observe`、`T_idle` 在 C2.8 的硬上限层注册——作为思考层运行参数
