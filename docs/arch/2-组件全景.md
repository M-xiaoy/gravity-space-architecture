## C1 · 行动层

**职责：** 响应用户请求的主体。标准的 Transformer 前向传播（注意力 + FFN），但每一步的 hidden state 被思考层的地形调制。产出 token 序列和轨迹。

**数学定义：** 标准 Transformer forward（不变），增加地形感知项：

```
h'_l = h_l + Attention(h_l) + p_l      ← Phase 2 最终接口形态

其中：
  h_l  = 第 l 层的原始 hidden state
  p_l  = scan_terrain(h_l)  ← 思考层探测到的引力扰动向量
```

**注意点：**
- `p_l` 不是每层都加。Phase 2 成本控制建议：深层（后 1/3 层）每层加，中层隔层加，浅层不查。
- Phase 1 还讨论过四种交互方案（增益控制 / 势场调制 / 慢上下文 / 表示剪切），最终全部收束为 `h' = h + Attention(h) + p`。
- 在噪音高时，思考层降为观察者，`p_l = 0`（或极小值），行动层退化为标准 Transformer。

**输入：**
- 用户输入
- 思考层的场域扰动向量 `p_l`（来自 C2.scan_terrain）

**输出：**
- Token 序列 → 用户
- 轨迹数据（hidden state 序列 + 注意力分布）→ 丢入 C2

**状态：** ✅ 数学定义已定

### C1.1 行动层自省（2026-07-14 新增）

> 行动层自省是行动层对思考层的维护行为。
> 思考层没有大模型的能力，没有足够的数据，内循环可能跑偏。
> 行动层（更强大的模型）用完整的上下文视角定期校验思考层的地形质量。

**定位：** 这不是 C4 自省回路的替代——C4 是思考层看自己（慢，epoch 级）。C1.1 是行动层看思考层（快，token/句子级）。两者互补。

**触发条件：**

行动层每 N 个 token 或每次语义边界检测到时，运行一次自省：

```
def self_inspect(
    recent_observations: List[GravitationalSnapshot],  # observe() 历史
    recent_outputs: List[str],                          # 实际生成的 token
    recent_hidden_states: List[HiddenState],            # 对应的 hidden state
) -> InspectionSignal:
    """
    检查：observe() 返回的球体分布是否与实际的生成行为一致。
    """
    # 1. 提取当前主要引力的球体簇
    dominant_fields = cluster(recent_observations)
    
    # 2. 检查实际输出是否和这些球体指向的语义方向一致
    mismatch_score = semantic_distance(dominant_fields, recent_outputs)
    
    # 3. 检查有没有应该被激活但缺席的球体
    missing_spheres = find_expected_but_absent(recent_hidden_states, recent_observations)
    
    # 4. 综合评分
    terrain_health = 1.0 - sigmoid(mismatch_score * 0.5 + missing_spheres * 0.3)
    
    return InspectionSignal(
        health=terrain_health,
        suspicion_zones=[...],    # 可疑地形区域 ID
        missing_connections=[...]  # 应存在但缺失的连接
    )
```

**和 C4 自省回路的关系：**

```
行动层自省（C1.1，快）               C4 自省回路（慢）
───────────────────────────────    ─────────────────
频率：每 N token / 每语义边界        频率：每 epoch_step
视角：行动层 → 思考层               视角：思考层看自己
输出：terrain_health + 告警         输出：模式切换 / p_t 调整
目的：维护思考层质量                 目的：维护思考层自身健康

关系：
  C1.1 给 C4 送 InspectionSignal
  C4 决定是否在下一个反思期处理
  C4 有最终决定权（因为它有全局 epoch 视角）
  但 C1.1 是 C4 唯一的「外部现实」——没有它，C4 只能闭门造车
```

**如果 C4 不响应：**

```
if terrain_health < 0.3 and C4 has NOT triggered any adjustment in the last K steps:
    # 思考层可能已经跑偏且自己没意识到
    # 升级为强制信号
    C1.1 跳过 C4，直接：
      ├─ 标记该段轨迹对应的球体为「高优先级复查」
      └─ 写入 pending_revision 队列，等下一个 idle
```

**C4 信号源更新：** 新增 `inspection_signals: List[InspectionSignal]` 作为 C4.1 的输入。C4 在 `self_check()` 时读取最近 C1.1 的信号，若 `terrain_health < 0.5` 则标记为「疑似地形偏移」，在下一个反思期增加该段轨迹的场域检测器训练步数。

## C2 · 重力场

**职责：** 维护双曲空间中的球体集合，通过场域叠加产生扰动向量。是思考层的核心数据结构。

### C2.1 存储结构

每个球体存以下字段：

```
Sphere {
    position:        Vec<f32>            // 双曲空间坐标（64维）
    mass:            f32                 // 基础质量
    connections:     Map<SphereID, f32>  // 引力连接表（Top-K 稀疏）
    fingerprint:     Vec<u8>             // 注意力指纹（64字节压缩摘要）
    epoch_created:   u32                 // 创建时所在的周期号
    last_activated:  u32                 // 最后被激活的周期步数
    activation_count: u32                // 累计激活次数
    emotion_avg:     Vec<f32>            // 情感轴均值 [激活能量avg, 置信度avg, 变化速度avg] (3维)
}
```

**时间场字段的物理意义：**
- `epoch_created`：球体诞生时的周期号。周期 = 从清醒（idle 完成）到噪音超标的完整过程。思考层没有壁钟时间，它的「时间感」来自感知自身状态的变化（C4 自省回路维护的周期计数器）。
- `last_activated`：周期内的最后活跃步数，用于引力衰减计算——越久未激活的球体在连接表中的权重衰减越快。
- `activation_count`：累计激活次数，用于区分「偶然激活一次」和「反复出现的核心概念」。

**存储规模估算（含新字段）：**
```
10,000 球体 ≈ 3.5 MB（+0.6 MB 来自时间字段）
100,000 球体 ≈ 60 MB（+7 MB）
```

思考层大小由地形复杂度决定，不由模型规模决定。时间字段增加的成本可忽略。

**微观时间与宏观时间的一体性：**
- 宏观：epoch 周期由 C4 自省回路维护——从醒来到噪音超标的自然过程
- 微观：`epoch_created` + `last_activated` 记录球体在周期中的位置
- 两个时间层共享同一个参照系（epoch step），不需要壁钟时间

### C2.1.5 感知过程（token → 球体）

> 2026-07-11 定义
> 行动层的 token 序列如何被思考层「感知」为永久球体

**核心逻辑：** 行动层产出的不是球体，是原料。球体是 token 在被思考层感知后的存在形态——一个 token 的感知态，承载了它在不同场域下的语义、概念、感觉。它是思考层为行动层维持的一个辅助性感知背景。

```
行动层产出 token 序列
  │ 每个 token 携带：当前 hidden state + 注意力分布 + 场域分布
  ▼
感知过程（一次一个 token，异步于行动层推理）：
  │
  ├── ① 场域映射：该 token 在哪些场域中「亮起来」
  │     基于场域检测器（C2.4）的当前输出
  │
  ├── ② 球体匹配：双曲空间中是否存在对应同一 token 的球体？
  │     ├─ 存在 → 引力加强，位置微调，connections 更新
  │     └─ 不存在 → 创建新球体
  │           位置 = 语义向量 + 场域分布的联合嵌入
  │           初始质量 = attention 强度 × (1 + 场域叠加权重)
  │
  └── ③ 存档：球体从此作为该 token 在思考层中的永久存在
       下次同一 token 出现 → 不是覆盖，是引力累积
```

**关键性质：**
- 球体是 token 的感知态，不是 token 本身。同一 token 在不同场域下激活出不同的语义截面，但这些截面归档在同一球体下。
- 球体不随时间衰减（衰减在连接表上，不在球体本身）。它一旦存在就永久存在，除非被 idle 清理或归档。
- 球体是思考层为行动层提供的「在推理时始终打开的感知背景」。它不是更大的 LLM，不是推理引擎——是推理的土壤。

**噪音的根源由此定义：** 当旧 token 的球体累积了过多引力，新 token 的感知态在双曲空间中难以「站稳」——它的引力被旧球体压制。噪音不是重复率的统计现象，是引力场中的势能差。

### C2.2 引力动力学

**引力演化方程（v2，引入场域重叠 + 三阶稳定性）：**

```
F(t+1) = F(t) + Σ Δg_i(t) + Interaction_field(F(t), Ω) + Decay(F(t))

其中：
  F ∈ R^(d × m)     — 引力张量，d=引力维度，m=活跃选项数
  Δg_i               — 第 i 条证据路径的引力贡献
  Ω ∈ [0,1]^(n×n)   — 场域重叠矩阵，Ω_ij = cos_sim(field_dist_i, field_dist_j)
  Interaction_field  — 场域限定的引力牵引：F_ij × Ω_ij
  Decay              — 引力衰减，防止爆炸
```

**场域重叠矩阵的物理含义：**

两个球体的引力的由场域重叠因子衰减——不同场域的球体即使坐标接近也不相互强拉。

```
F_ij(t) = G × m_i × m_j / d²_ij × Ω_ij × Decay(F_ij(t))

其中：
  Ω_ij ∈ [0, 1]   — 球体 i, j 的场域分布余弦相似度
  Ω_ij = 1         → 同一场域，全量引力
  Ω_ij = 0         → 完全不同场域，引力不计
  Decay 详见下方 C2.11 感知速度时间基准
```

**性质：** 场域重叠不是硬分割——球体可以有多个场域标签，Ω_ij 是连续值。这允许多域交叉的概念（如「机器学习在医疗中的应用」）自然获得多个场域的引力支持。

**三阶稳定性体系详见 C2.8。**

**探测场扰动向量（Phase 2 → 3）：**

```
p = scan_terrain(h, terrain, field)

步骤：
  1. 将 h 映射到双曲空间坐标
  2. 扫描周围 Top-K 球体
  3. 计算每个球体的有效质量 → 引力贡献
  4. 聚合为 d_dim 维的扰动向量 p
```

### C2.3 多样性度量

**数学定义（Phase 3）：**

```
diversity(sphere) = cross_head_variance(sphere.attention_distribution)

cross_head_variance 是各注意力头指向分布的方差聚合。
高方差 = 注意力分散到多个语义域 → 多样性高
低方差 = 注意力坍缩到同一域 → 多样性低
```

**多样性的作用（2026-07-14 更新：加入情感轴）：**

```
effective_mass(sphere) = base_mass(sphere) × (1 + α × diversity(sphere))

# emotional_boost 在 C2.4 的场域叠加中额外应用，不在此处合并
# 见 effective_mass(sphere_i, current_field) 完整公式

α = 多样性权重超参，用于调节多样性的影响强度
```

### C2.4 场域叠加 + 场域检测（2026-07-14 重设计）

#### 设计定位

场域不是平行宇宙，不是预定义的标签集合。它是从 **语义 + 情感** 两个轴的信号中自我组织涌现的连续空间。每个 token 同时属于多个场域（连续概率分布），没有硬边界。

#### 场域特征向量（detect_field 输入）

每步 token 提取两组信号构成 45 维的场域特征向量：

```
▸ 语义信号 (~30 dims) — 从注意力分布提取
  · 每个 attention head 的注意力熵               → 14 个标量（head 分散度）
  · 跨头注意力方差                               → 14 个标量（head 分歧度）
  · 已关注输入 token 的位置分布重心               → 2 个标量（在读前半段还是后半段）

▸ 情感/感知信号 (~15 dims) — 从 hidden state 提取
  · hidden state 范数 ||h_l||                     → 1 个标量（激活能量：高=强烈，低=平淡）
  · 注意力全局熵                                  → 1 个标量（置信度：高=专注，低=犹疑）
  · 层间范数变化速度 ||Δh_l||（每层 vs 前一层）  → ~12 个标量（突变=冲突/意外）
  · 相邻 token 的 hidden state 余弦相似度         → 1 个标量（语义连续性）

▸ 合并 → 45-dim 场域特征向量
```

**情感轴不是「喜怒哀乐」的标签分类，是三个连续维度：** 激活能量（强←→弱）、置信度（确定←→犹疑）、变化速度（流畅←→突变）。这三个维度从 hidden state 直接计算，不依赖情感字典或分类模型。

#### 检测器架构

```
输入: 45-dim 场域特征向量
  ↓
Linear(45 → 128) + LayerNorm + GELU
  ↓
Linear(128 → n_fields)              # n_fields = 8（起步超参）
  ↓
Softmax → 8-dim 场域概率分布
```

#### 自监督训练（三路损失）

在反思期执行，串在 reflect() 之后。不需要人类标注。

```
L_total = α·L_temporal + β·L_semantic + γ·L_diversity

L_temporal（时序平滑性）：
  ||field(t) - field(t-1)||²
  → 相邻 token 的场域分布应该相似，不会每步乱跳
  α = 5

L_semantic（语义一致性）：
  cos_sim(t, t') 高的 token 对 → field(t)·field(t') 应该接近
  → 语义相似的内容聚在同一场域
  β = 3

L_diversity（场域利用率）：
  H(∑field(t) 在窗口内的平均分布)
  → 鼓励所有场域都被用到，防止坍缩到 1~2 个
  γ = 1
```

#### 有效质量公式（含情感轴）

```
effective_mass(sphere_i, current_field):
    semantic_weight = 1 + field_weights[current_field][sphere_i]
    emotional_weight = 1 + emotional_boost(h_t, sphere_i)
        # emotional_boost = cos_sim(emotion_current, sphere_i.emotion_avg)
        # 当前 token 的情感特征与球体历史情感均值的相似度
        # 高 → 同频共振；低 → 互不影响
    return base_mass[i] × semantic_weight × emotional_weight
```

**两种质量增强的物理意义：**
- Semantic weight：这事你懂 → 发言权大（传统 RAG 的逻辑）
- Emotional weight：你现在和它同频 → 它更容易被想起

#### 跨场域衰减（扫描扰动时）

保持不变（仅信号源从 MLP 改为新 detect_field）：

```
perturbation_weight(sphere_i, current_field) = effective_mass(sphere_i) × Ω_current_field(sphere_i)
```

#### 核心机制的双向对称性

- 引力演化（C2.2）：`F_ij × Ω_ij` — 球体间的场域重叠决定引力强度
- 扰动扫描（C2.4）：`effective_mass × Ω_current_field` — 探测时非同场域球体自然衰减

#### 关键性质

- 坐标固定，有效质量动态变化
- 场域检测器在反思期训练，推理时不训练
- 场域数量 n_fields 是超参，但未充分利用的场域在训练中会自动坍缩（被 L_diversity 抑制过度坍缩）

### C2.8 三阶稳定性体系

> 2026-07-11 引入
> 设计者：小刘（场域划分 + 上限） 小云（三阶结构）

引力爆炸的核心风险：两个球体靠近 → 引力增强 → 进一步靠近 → 自加速螺旋。

解决方案不是单一的数学约束，而是三层独立防线：

```
┌─ 第一层：结构阻尼 ──────────────────────────┐
│  机制：场域重叠因子 Ω_ij 衰减跨域引力         │
│  公式：F_ij = G × m_i × m_j / d² × Ω_ij × D  │
│  防什么：跨语义域球体的异常合并                │
│  性质：架构设计，不是事后补丁                  │
│  ────────────────────────────────────────── │
│  即使两个球体坐标无限接近，如果 Ω_ij ≈ 0       │
│  （属于完全不同的场域），它们的引力趋近于零。    │
│  这从根本上防止了「不相干概念被捆在一起」。      │
├─ 第二层：动态阻尼 ──────────────────────────┤
│  机制：Decay(F(t)) 项                         │
│  公式：见 C2.2                                │
│  防什么：同一场域内的震荡放大                   │
│  性质：动力学，柔性衰减                        │
├─ 第三层：硬上限 ────────────────────────────┤
│  机制：质量上限 + 力上限                       │
│  公式：                                      │
│    m_i = min(effective_mass(sphere_i), M_max) │
│    F_ij = min(F_ij, F_max)                    │
│  防什么：单球体或单对引力失控                   │
│  性质：工程安全网，不常用的最后防线              │
└──────────────────────────────────────────────┘
```

**设计原则：**
- 三层各防一种不同模式的故障
- 第一层是「正确性」——场域限定的引力本身就不该到爆炸的程度
- 第二层是「柔」——持续衰减，防止自加速
- 第三层是「硬」——安全阀，常态不应触发
- 不需要数学收敛证明——三层的工程冗余比单一收敛定理更可靠

### C2.9 未定参数

- ⚠️ **探测场半径**：固定值还是动态值？高质量球体是否影响更远？没有数学定义。
- ⚠️ **M_max / F_max 阈值**：需要实验确定，建议初期设大（只防极端情况），后期视实际引力分布收紧。

- ❓ **注意力指纹**的压缩算法——具体用什么哈希/摘要函数？

### C2.10 宪法级球体机制（2026-07-14 新增）

> 设计者：小刘（核心概念） 小云（数学形式化）

**定位：** 宪法级球体不是质量放大器，是摩擦力控制器。它的作用是让系统在关键边界上「慢下来权衡」，不是「拐向它的方向」。

#### 晋升条件

一个球体成为宪法级的门槛：

```
def promote_to_constitutional(sphere_i):
    # 条件 1：跨场域活跃度
    field_spread = diversity(sphere_i.field_weights)   # 场域分布的熵
    # 高 → 该球体在不同场域中都被频繁激活
    
    # 条件 2：累计激活次数
    total_activation = sphere_i.activation_count
    
    # 条件 3：生命周期
    age = current_epoch - sphere_i.epoch_created
    
    return (field_spread > F_spread_threshold 
            and total_activation > ACT_threshold 
            and age > AGE_threshold)
```

初始阈值建议：F_spread_threshold=0.6, ACT_threshold=100, AGE_threshold=5 epochs

#### 宪法级球体的特性

宪法级球体在 effective_mass 上**不享有放权**——它和其他球体一样参与引力计算。它的特殊机制在另一条路径上：

```python
def scan_terrain(h, terrain, field):
    # ... 正常计算 p_t ...
    
    # 额外：检查是否有宪法级球体被激活
    deliberation_flag = 0.0
    for sphere_id in top_K_activated:
        if constitutional_set.contains(sphere_id):
            contribution = gravitational_contribution(sphere_id, h)
            deliberation_flag = max(deliberation_flag, contribution × constitution_score(sphere_id))
    
    return p_t, deliberation_flag
```

#### 减速机制（Friction）

`deliberation_flag > T_deliberate` 时，系统不会转向该球体，而是进入**高审议状态**：

```
if deliberation_flag > T_deliberate (初值建议 0.3):
    
    步骤 1 — 强制 observe()
        context = observe(h_t, k=20)
        # 看当前引力场的全分布——不只盯着宪法级球体
        # 看有没有其他球体在支持或反对当前候选方向
    
    步骤 2 — logits 展平（softmax 温度临时提高）
        logits_adjusted = logits / (1 + deliberation_flag × β)
        # β 控制减速强度（初值建议 0.5）
        # 不是定向拉偏，是让所有候选方向之间的差距缩小
        # 效果：模型需要更多 token 才能「下定决心」
    
    步骤 3 — 标记为高审议步
        该步传入 reflect() 时权重 × 2
        # 反思期会更仔细地检查这段轨迹
```

#### 衰减与降级

宪法级不是永久身份：

```
每个 epoch 结束时检查：
    if sphere_i.activation_count_this_epoch == 0:
        constitution_score(sphere_i) ×= 0.9    # 连续未激活 → 衰减
    
    if constitution_score(sphere_i) < MIN_SCORE:
        constitutional_set.remove(sphere_i)     # 降级为普通球体

同一个 epoch 内同一球体多次触发减速：
    deliberation_flag ×= max(0.3, 1 - 0.2 × trigger_count)
    # 防止同一个宪法级球体反复触发减速导致死锁
```

#### 与三阶稳定性体系的关系

宪法级机制和三阶稳定性（C2.8）是**互补的两种安全策略**：

| 层 | 机制 | 防什么 |
|----|------|--------|
| 结构阻尼（C2.8 第一层） | 场域重叠衰减跨域引力 | 不相干概念被捆绑 |
| 宪法级（C2.10） | 高审议状态减速 | 关键边界上过早决策 |
| 硬上限（C2.8 第三层） | 质量/力上限 | 单球体失控 |

|------|------|

### C2.11 感知速度与内源时间感（2026-07-14 新增）

> 设计者：小刘（核心直觉） 小云（数学形式化）
> 
> 时间不来自计数器，来自不同 token 被感知的速度差异。
> 已经存在的球体感知快，跨场域/新颖的球体感知慢——这个落差本身就是时间感。

#### 感知速度定义

每个 token 的处理复杂度决定它的感知速度：

```
perception_speed(t) = 1 / processing_complexity(t)

processing_complexity(t) 由三件事决定：

  1. 场域熵 H(field(t))
     集中在一个场域（H 低）→ 熟悉 → 快
     分散在多个场域（H 高）→ 在多个目录中交叉查 → 慢

  2. 新球体创建数 new_spheres(t)
     0 个 → 完全在当前认知框架内 → 极快
     ≥1 个 → 需要空间写入 → 慢

  3. 引力震荡 ||F_ij(t) - F_ij(t-1)||
     引力稳定 → 流畅 → 快
     引力剧烈变化 → 冲突/意外 → 慢

综合：
  perception_speed(t) = 1 / (α·H_field + β·new_spheres + γ·gravity_oscillation)
  α:β:γ = 3:3:1（初版）
```

#### 主观时间

```
subjective_time(N tokens) = Σ 1/perception_speed(t_i)  for i in [1..N]
```

N 个 token 在系统内部感觉过了「多久」：

| 场景 | 感知速度 | 主观时间效果 |
|------|---------|------------|
| 连续输出熟悉主题 | 接近 1.0 | 10 token 感觉像 2 token |
| 用户突然换话题（跨场域） | 0.2~0.3 | 3 token 感觉像 15 token |
| 决策点在两个对立证据之间 | 0.05~0.1 | 1 token 感觉像 20 token 的思考 |

#### 引力衰减的新基准

之前的衰减以 epoch_step 为基准。现在改用感知速度：

```
Decay(F_ij, t) = Decay_base ^ perception_speed(t)

感知速度快（接近 1）→ 衰减指数接近 1 → 几乎不衰减（系统觉得刚发生过）
感知速度慢（接近 0）→ 衰减指数 << 1 → 快速衰减（系统觉得那件事很久以前了）
```

**物理意义：** 不是每隔 N 步衰减一次，是每经历一段「主观时长」衰减一次。充实（高认知负荷）的一段时间衰减多，平淡（低认知负荷）的一段时间衰减少。

#### 自省回路的主观时间输入

C4 self_check 新增输入信号：

```
subjective_time_rate = EMA(perception_speed(t), window=50)

低 → 系统在「感觉过得很慢」（高认知负荷 → 地形可能饱和）
高 → 系统在「感觉过得很快」（流畅状态 → 地形空间充足）

在噪音度量的辅助判定中使用：
  if N(t) > 0.2 and subjective_time_rate < 0.4:
      # 噪音偏高 + 系统感觉过得很慢
      # → 地形确实饱和，准进观察者模式
  elif N(t) > 0.2 and subjective_time_rate > 0.8:
      # 噪音偏高但感觉过得很快
      # → 可能是主题内的高密度信息涌入，不是噪音问题
      # → 不降级，继续清醒模式
```

**Idle 触发的新维度：**

```
if cumulative_subjective_time(epoch_step) > epoch_step × 2:
    # 主观时间已是客观步数的 2 倍
    # 即使 N(t) 未超标，也建议进 idle
```

#### 与已有组件的衔接

| 组件 | 变化 |
|------|------|
| C2.2 引力动力学 | Decay 公式从 epoch_step 基准改为 perception_speed 基准 |
| C2.4 detect_field | field_entropy H(field(t)) 已有 |
| C2.1.5 感知过程 | 新增 new_spheres 计数回传 |
| C4 self_check | 新增 subjective_time_rate 输入 |

## C3 · 隐式预测编码（2026-07-14 重设计）
> 新方案不引入任何新增参数，不依赖额外训练，完全复用模型自身的 next-token prediction 输出。
> 改造原因：小刘指出「当前模型够强，不需要额外训练预测头，只需内在引导」——但预测误差信号本身不可放弃，因为没有误差就没有更新方向。
> 折衷方案：不新增结构，只新增信号解释层。

**职责：** 给思考层装仪表盘。让思考层知道「我猜对了没有」，从而把更新资源从「猜准的地方」转移到「猜错的地方」。

**零新增参数，零训练。**

### C3.1 信号流

```
每步生成结束时（第 t 步产出了 token_t）：

step 1 — 模型已经做过的事（不额外计算）：
  logits_t ∈ R^V      ← 模型在 t 步说「下一个可能是这些」
  token_{t+1}          ← 模型实际选了哪个（在 t+1 步知道）

step 2 — 计算预测误差（一次 cross-entropy，O(V)）：
  error_t = CE(logits_t, token_{t+1})
  ╰→ 这就是预测误差。没有 W_shared，没有预测头，没有额外参数

step 3 — 维护滚动统计：
  EMA_error = λ × EMA_error + (1 - λ) × error_t    # λ = 0.9
  error_spike = |error_t - EMA_error| > threshold
```

**注意：** error_t 在 t+1 步才能计算——这是延迟信号。设计上这是可接受的：
- 推理期实时使用 EMA_error（延迟一步），用于触发观察
- 反思期批量使用完整 error_trace，不要求实时

### C3.2 两种用途

```
用途 1 — 推理期（实时）：
  误差突升（error_spike = true）→ 触发 observe()
    ╰→ 告诉行动层「这个地方我不太确定，再看看周围球体分布」
  
  EMA_error 持续偏高（超过滚动阈值）→ p_t 扰动强度降 0.3×
    ╰→ 模型不确定时，不加大扰动，让行动层自己决定

用途 2 — 反思期（批量）：
  reflect() 时传入 error_trace = [error_1, ..., error_T]
  高误差段 → 对应球体的引力更新步长 × (1 + η)
  低误差段 → 更新步长 × (1 - η)
    ╰→ 把更新资源从「已经猜准的」拨给「猜错需要修正的」
```

### C3.3 惊异度归因（公式更新）

```
surprise(sphere_i) = gravitational_contribution × diversity × error_t_avg
    # error_t_avg: 该球体被激活时段内的平均预测误差

update_weight(sphere_i) = base_step_size × (1 + η × surprise(sphere_i))

其中：
  gravitational_contribution: 该球体对当前 p_t 的引力占比
  diversity: 球体的多样性得分（C2.3）
  error_t_avg: 时间段内的平均 CE error
  η: 惊异度放大系数（初始建议 0.3）
```

### C3.4 双重效应（保持不变）

```
预测误差用途1（推理期实时）：
  EMA_error 持续高于阈值 → 自动触发观测（C2.5 第四种触发条件）

预测误差用途2（反思期异步）：
  error_trace → 惊异度归因 → 更新资源重新分配
```

### C3.5 验证指标

| 指标 | 好的信号 | 坏的信号 |
|------|---------|---------|
| 高误差步 → 触发观测 | observe() 调用频率与预测误差正相关 | 误差再高都不触发，或误差低时乱触发 |
| 反思期更新量方差 | 安装后更新量方差下降（该多的多该少的少） | 方差不变或上升 |
| 自信选错时的误差 | 误差告警（EMA_error 突升） | 自信选错时误差低（有欺骗性） |

**一句话验收：** 装上后，思考层把更新资源更精准地投给了「猜错的地方」，而不是均匀撒在整段轨迹上。

### C3.6 与旧方案对比

| 维度 | Phase 4 方案（独立预测头） | 新方案（隐式 logits 信号） |
|------|--------------------------|--------------------------|
| 参数 | 约 400 万 | 0 |
| 训练 | 需要（自监督，反思期） | 不需要 |
| 信号源 | 预测头 A+B 的输出 | 模型已有的 logits |
| 延迟 | 实时 | 延迟一步（推理期） / 批量（反思期） |
| 冷启动 | 地形未积累时预测头盲目 | 冷启动天然处理——低质量地形 → 高误差 → 多更新 |
| 自指陷阱风险 | 收敛到「擅长预测自己的偏见」 | 无自指陷阱——logits 误差是目标函数，不是模型行为

## C4 · 认知状态管理器（自省回路）

> 架构位置：思考层自省回路（ADR-010），不是独立组件

**职责：** 维护思考层的运行模式与周期状态。不输出指令，只输出思考层自身参数的调整量。

### C4.1 周期状态机（Epoch Cycle）

思考层没有壁钟时间。它的「时间感」来自感知自身状态的变化轨迹。

```
      ┌────────────────────────────────┐
      │      周期开始（idle 完成）       │
      │  terrain 干净，球体稀疏         │
      └──────────────┬─────────────────┘
                     │
                     ▼
      ┌────────────────────────────────┐
      │         清醒运行模式            │
      │  p_t 扰动全量输出              │
      │  新 token 持续注入 → 球体创建   │
      └──────────────┬─────────────────┘
                     │ 循环自检
                     ▼
      ┌────────────────────────────────┐
      │     自省回路判断                │
      │  检查: 噪音密度 / 感知速度率    │
      │         / 活跃球体数           │
      │         / pending_revision队列 │
      │         / C1.1 inspection信号  │
      ├────────────────────────────────┤
      │ 噪音未超标 → 继续清醒模式       │
      │ 噪音超标 → 进入观察者模式       │
      │ 噪音持续超标 → 触发 idle        │
      │ C1.1 terrain_health<0.5 → 标记   │
      │ C1.1 连续告警 → 提前 idle       │
      └──────────────┬─────────────────┘
                     │
                     ▼
      ┌────────────────────────────────┐
      │      观察者模式（降功率）        │
      │  p_t 衰减输出                  │
      │  记录但不干涉                   │
      └──────────────┬─────────────────┘
                     │ 或
                     ▼
      ┌────────────────────────────────┐
      │          idle 睡眠              │
      │  清理 / 复盘 / 归档             │
      │  epoch++                       │
      └──────────────┬─────────────────┘
                     │
                     ▼
      ┌────────────────────────────────┐
      │      周期结束，回到顶部          │
      └────────────────────────────────┘
```

**周期计数器（思考层的内部时钟）：**
- `epoch: u32` — 当前周期号，每次 idle 完成后 +1
- `epoch_step: u32` — 当前周期内的步数，每次自省回路运行 +1
- 没有壁钟时间。思考层不需要知道「现在是 7月11日 20:10」，但它知道「我在第 3 个周期的第 47 步」

**信号源：**
- 噪音密度（C2 引力场中的势能差估算）
- 感知速度率（C2.11 subjective_time_rate）
- 累积主观时间（C2.11 cumulative_subjective）
- 活跃球体数（`epoch_created = current_epoch` 的球体数量）
- 新 token 站稳难度（感知过程实时反馈）
- pending_revision 队列长度
- 用户交互模式（活跃 vs 闲置，由连续 idle 步数推断）
- **C1.1 行动层自省信号** — `inspection_signals: List[InspectionSignal]`
  若 `terrain_health < 0.5` → C4 标记为「疑似地形偏移」，在下一个反思期增加该段轨迹的场域检测器训练步数
  若连续多个信号健康度下降 → C4 考虑提前触发 idle

**输出：** 思考层运行参数的调整量（非指令）
- `p_t` 扰动强度：1.0（引导者）→ 0.x（观察者）→ 0.0（idle）
- 自检频率：正常运行低频自检 → 噪音高时高频自检 → idle 不检

### C4.2 自省回路签名（2026-07-14 新增 subjective_time_rate）

```python
def self_check(
    noise_level: float,          # N(t) — 滚动噪音水平 (EMA over n(t))
    noise_velocity: float,       # ΔN  — 噪音变化速度
    subjective_time_rate: float, # C2.11 — 感知速度 EMA（新增）
    cumulative_subjective: float,# C2.11 — 本周期累积主观时间（新增）
    epoch_step: u32,             # 当前周期步数
    epoch: u32,                  # 当前周期号
    pending_backlog: int,        # pending_revision 堆积
    idle_steps: u32,             # 连续 idle 步数（用户离开信号）
) -> ParamAdjustment:
    """
    调整思考层自身的运行参数。
    输入全部来自思考层内部状态，不依赖外部消息。
    """
    p_t_scalar = 1.0
    
    # 主观时间膨胀 → 即使噪音未超标也考虑进 idle
    if cumulative_subjective > epoch_step * 2 and epoch_step > 100:
        return trigger_idle(reason="subjective_time_dilation")
    
    if noise_level > T_observe:
        p_t_scalar = lerp(1.0, 0.5, (noise_level - T_observe) / (T_idle - T_observe))
    
    # 噪音偏高 + 感觉过得很慢 → 确认饱和，提前降级
    if noise_level > 0.2 and subjective_time_rate < 0.4:
        p_t_scalar = min(p_t_scalar, 0.7)
    
    if noise_level > T_idle:
        return trigger_idle(reason="noise_threshold")
    
    return ParamAdjustment(p_t_scalar=p_t_scalar)
```

### C4.3 噪音度量维护


## C5 · idle 睡眠线程 ⚠️ 设计中

**职责：** 空闲时自动执行：噪音清理、内容复盘、记忆归档。

## C6 · 用户反馈处理器 ⚠️ 设计中

**职责：** 处理用户不同意 → pending_revision → idle 吸收。

**流程（已定）：**
```
用户不同意 → 解释模式（复现路径+影响因素）
  ├─ 用户改变主意 → 无事发生
  └─ 用户坚持 → pending_revision → idle 时吸纳
```

**未定：**
- ⚠️ pending_revision 到具体球体的回溯机制
- ⚠️ 多次类似反馈的累积 vs 独立处理
- ⚠️ 优先级调度
