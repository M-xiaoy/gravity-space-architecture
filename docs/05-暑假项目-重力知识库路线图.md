# 暑假项目 · 重力知识库

> 记录于 2026-07-09，与小刘共同敲定
> 
> 原则：双曲高度=0，FAISS做检索，重力空间做路由，多样性和场域由用户控制

---

## 一、要造一张什么牌

本地知识问答系统。不是命令行玩具——是真正可部署、有 Web 界面、别人能直接拿来用的知识库工具。

**跟所有现有知识库工具的差异：**
- 检索结果不是按相似度一路排到底——经过重力空间的路由层：跨来源优先、同来源去重、场域加权
- 用户能控制场域权重——上传时打标签，查询时调权重
- 底层靠 FAISS 做速度，上层靠重力空间做质量

**为什么是这张牌：**
- 市场需求真实存在——每家公司都想把内部文档接上大模型
- 差异化明确——只做知识库的公司不会做重力空间路由层
- 可演示——在面试时打开浏览器，现场传一份文档，问一个问题，展示中间检索过程

## 二、架构（已定型，见架构图）

```
用户
  ↓
交互层：文件上传 / 问答界面 / 会话管理
  ↓
知识管线：文件解析 → 切片器 → 嵌入器 → 来源登记
  ↓
存储层：FAISS 向量索引 + 重力空间球体库
         │ FAISS 只管存向量、做粗搜 Top-100
         │ 球体库管元数据、质量、场域、来源指纹
  ↓
重力空间检索层：场域检测 → 多样性排序 → Top-5 输出
  ↓
生成层：上下文构建 → Ollama LLM → 回答
```

**已定型的设计决策（不可轻易改）：**

| 决策 | 内容 |
|------|------|
| 双曲空间高度 | v1 全部置 0，FAISS 当 2D 用 |
| 存储双轨 | FAISS 存向量做粗搜 + 球体库存元数据做路由 |
| 多样性定义 | 来源多样性 = 源文件分布广度 |
| 场域权重 | 用户手动设定，不上传打标签 |
| 连接表更新 | 轮询 + 触发式（查询时高频命中的球体更新邻居引力） |

## 五、关键接口定义

### 球体数据结构

```python
@dataclass
class Sphere:
    id: str                     # 唯一标识
    text: str                   # 原文片段
    vector: np.ndarray          # 嵌入向量（768d / 1024d）
    source_file: str            # 源文件名
    source_type: str            # 场域标签（用户上传时打）
    mass: float = 1.0           # 基础质量
    diversity: float = 0.0      # 多样性（来源分布广度）
    effective_mass: float = 1.0 # mass × (1 + diversity)
    connections: dict = None    # {sphere_id: 引力强度}
```

### 检索输出

```python
@dataclass
class RetrievalResult:
    sphere: Sphere
    faiss_score: float          # FAISS 原始相似度
    field_weight: float         # 场域匹配度（0-1）
    diversity_bonus: float      # 多样性加成（0-1）
    final_score: float          # 最终排序分
```

### scanner.py 的检索流程

```python
def search(query_vector, top_k=5):
    # 1. FAISS 粗搜 Top-100
    faiss_results = faiss_store.search(query_vector, n=100)
    
    # 2. 检测场域
    current_field = field_detector.detect(query_vector, faiss_results)
    
    # 3. 多样性排序
    candidates = [sphere_store[faiss_id] for faiss_id in faiss_results.ids]
    reranked = diversity_sort(candidates, current_field)
    
    # 4. 返回 Top-5
    return reranked[:top_k]
```
