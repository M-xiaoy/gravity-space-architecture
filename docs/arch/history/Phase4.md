> 2026-07-09，推演会议的第二场
>

---

## 先讲清楚这个架构在做什么

写到这里，需要把三期以来没明说的事情说清楚。

重力空间架构不是在改进通用大模型。通用大模型的训练数据来自全互联网，学习目标覆盖所有用户的分布——那是另一个故事。

这个架构回答的是另一个问题：**如果模型只服务于一个人，它的架构应该长什么样？**

当思考层只为一个用户积累地形时，闭环不是偏差放大器——是**个性化加速器**。每轮对话让地形更贴合那个人的习惯。场域权重由用户亲手设定。多样性度量的是他一个人的知识来源跨度。

如果思考层和一个用户持续对话一年，它的引力分布会渐渐收敛到那个人的认知模式上。不是通过微调——是通过持续的地形更新，在不重新训练模型的前提下，让模型越来越适应一个人的表达方式。

三期以来的所有组件——地形感知、场域叠加、多样性度量、观测机制——都是个人化模型的配件。这篇接上最后一个缺失的配件：预测编码。

---

## 从 Phase 3 的边界出发

Phase 3 结束时，架构有了六个互相咬合的机制：地形感知、场域叠加、多样性度量、观测、存储结构、参数量验证。逻辑链闭合了。

但有一个组件没有落地。

思考层做的是**事后归纳**——收取轨迹、更新地形，像个历史学家。它知道发生了什么，知道地形长什么样，但它不知道「如果你往这个方向走，你会有什么感觉」。

历史学家不预测。预测是工程师的事。

## 为什么需要预测编码

Phase 2 提出了一个自监督训练目标：**给定当前重力场状态，能否预测行动层下一步的偏置？**

Phase 3 的所有组件——地形、场域、多样性、观测——都是为这个目标铺设的基础设施。但预测的想法本身被放在「下一阶段」的篮子里，没有被工程化。

原因很简单：我那时不知道预测编码在重力空间中的具体形态应该是什么。

这篇解决这个问题。

## 架构直觉：地形已经在做隐式预测了

先注意一个被我忽略了很久的事实。

思考层维护的地形——双曲空间 + 球体——不是一个静态数据库。当行动层站在 `h_t`，扫描周围球体的引力，得到扰动向量 `p_t`，这个 `p_t` 已经编码了「根据你当前的位置，接下来最相关的概念分布」。

**这不是别的什么——这就是一个隐式的预测。** 地形本身就是一个对「世界如何演化」的预期模型，只是没有显式化给自己看。

预测编码要做的，就是在这个隐式预测上搭一个仪表盘：让思考层能知道「我猜对了没有」。

一个认知系统只有在有预期的时候，才能被「意外」。被意外 = 预期和实际的差额 > 0。没有预期，所有输入权重相等——这就是当前标准 Transformer 的状态。

## 加三样东西

### 1. 预测头

一个轻量的自监督头，附在思考层上。它不参与日常推理——只在反思期被激活。

**输入共享层：**

```
x_t = concat(h_t, p_t, f_t)
z_t = W_shared · x_t + b_shared   # 压缩到 256 维

h_t: 行动层当前步的 hidden state
p_t: 当前探测到的引力扰动向量
f_t: 当前场域分布
```

压缩到 256 维后分叉成两个预测头。

---

**预测头 A（状态预测）：**

目标：根据当前地形+位置+场域，预测行动层下一步的 hidden state 变化方向。

```
Δh_pred = W_A · z_t + b_A
loss_A = ||(h_{t+1} - h_t) - Δh_pred||²
```

物理上它回答：「根据我脚下这片地形，我预期下一步的海拔变化是这个方向——实际走的跟预期一致吗？」

如果 loss_A 小，说明地形在这片区域预测准确。如果 loss_A 大，意味着地形需要更新——这里发生了我没料到的事。

---

**预测头 B（场域稳定性预测）：**

目标：预测接下来 N 步内，场域分布的变化幅度。

```
f_change_pred = W_B · z_t + b_B    # 输出是标量
loss_B = (||f_{t+N} - f_t|| - f_change_pred)²
```

场域稳定性不是二分类（变/不变）——是连续值。这个信息量比二分类多了一个数量级。它让思考层知道「不仅是变了，而且变的幅度跟我想象的差多少」。

比如从逻辑推理切换到闲谈，场域变化大；在同一个话题内深入，场域变化小。预测头 B 学的是这两种场景的区分。

---

**总损失：**

```
total_loss = 5 × loss_A + 1 × loss_B
```

α=5, β=1 的比值不是随便选的——loss_A 的信号比 loss_B 干净稳定，天然应该权重高。这个比值不事先固定，给思考层在反思期自己自适应调整：如果发现 loss_B 一直抖（场域预测确实难），β 自动降低；如果 loss_B 收敛稳定，β 可以爬升。

这叫**损失自适应加权**。

---

### 2. 误差计算器

反思期做的事情很简单：

```
遍历一段轨迹，对每一步计算：
  error_t = 5 × ||Δh_pred - Δh_actual||² + 1 × (field_change_pred - actual)²

如果 error_t ≈ 0：这片区域的地形已经准了，更新步长缩减
如果 error_t 高：这片区域我猜错了，更新步长放大
```

---

### 3. 惊异度归因

误差知道了，下一个问题——**哪个球体该为这个误差负责？**

就近归因的思路：引力贡献最大的球体背锅。但这有个盲区——只看距离不看语义。

如果一个球体被多条独立推理路径同时支撑（高多样性），然后它出错了——这意味着不是某一条路径抽风，而是整个地形在那个区域存在**系统性盲区**。这比单路径出错可怕多了，也应该被更认真地对待。

于是有了第三条归因路径：**惊异度归因。**

```
surprise(sphere_i) = gravitational_contribution × diversity × error_t
```

- **gravitational_contribution** — 这个球体对当前 `p_t` 贡献了多少引力（就近归因的那一半保留着）
- **diversity** — 多条独立路径支撑的球体出错了，比单一路径出错更「惊异」
- **error_t** — 预测头实际产生的误差大小

更新时：

```
update_weight(sphere_i) = base_step_size × (1 + η × surprise(sphere_i))
```

高惊异度的球体在反思期分到更多的更新资源。不是惩罚——是**注意力重新分配**。整个反思期的计算量没变，只是把资源从「已经猜准的地方」转移到了「猜错的地方」。

## 预测误差的双重效应

预测误差在架构中有两个独立的用途，路径不同：

**路径1（推理期实时）：误差持续异常 → 自动触发观测**

原本观测的触发条件有三个：注意力熵增、语义边界、首次收敛。现在有了第四个：**预测误差持续高于阈值。**

这形成了一个主动-被动互补的触发对：
- **观测（主动）**：行动层说「等等，我得看看周围」
- **预测误差（被动）**：思考层默默发现「这里我没猜对」

两种意外源。就像你走在路上：
- 主动回头环顾四周 = 观测
- 脚下踩到一个坑跌了一下 = 预测误差

---

**路径2（反思期异步）：误差加权 → 地形更新**

反思期做三件事：

| 步骤 | 原有/新增 | 作用 |
|------|----------|------|
| 1. 频率+多样性更新 | 原有 | 轨迹激活的球体频率+1，注意力评估多样性 |
| 2. 位置/连接表微调 | 原有 | 同时激活的球体互相拉近 |
| 3. 惊异度引导的加权更新 | 新增 | 高误差区域的球体更新步长放大 |

第三步不改变前两步的逻辑——只调整优先级。误差大的区域多分资源，误差小的区域少分。总更新计算量不变。

## 预测头的参数规模

预测头加编码层的总参数约 400 万——共享编码层 300 万 + 两个预测头各约 50 万。

400 万参数，只在反思期激活。日常推理时完全静默。

作为一个对比：行动层（7B 模型）在推理时每步都要调动 70 亿参数。思考层本身的地形操作也不需要大量参数。加到 400 万仅仅是为反思期增加了一个轻量训练回路——相对于架构的增量收益，这个成本在可接受范围内。

## 这个设计对已有机制的影响

我验证了每个已有机制，确认新机制不冲突、不替代、可联动：

| 已有机制 | 预测编码的关系 | 冲突？ |
|---------|--------------|-------|
| 观测 | 对偶关系。观测是主动回头看。预测误差是被动被意外。一个主动一个被动，正好覆盖认知闭环的两个扇区 | 不冲突 |
| 场域叠加 | 预测头 B 直接为场域机制增加了自监督训练信号。场域越稳定 → 预测越准 → loss_B 越低。场域越不稳定 → 预测越不准 → loss_B 越高 → 场域识别器收到反馈 | 不冲突，可联动 |
| 多样性 | 惊异度归因中的 diversity 项让多样性同时承担了两个角色：引力中的权重（Phase 3）和归因中的权重（Phase 4）| 不冲突，可复用 |
| 被动/主动双模式 | 预测误差可以作为主动模式的自动触发信号。误差持续高 → 系统自动从被动切主动 | 可联动 |
| 注意力指纹 | 指纹是多样性度量的输入。预测误差是另一维度。两个信号一起决定球体的更新优先级 | 不冲突，可合并 |

没有需要修改的已有机制。预测编码在 Phase 3 的完整基础上叠加，不破坏任何已定型的设计。

## 整个架构目前的全景

```
行动层每走一步 → 思考层暗暗产生预期（预测头）
                  ↓
反思期发生三件事：
  1. 频率/多样性更新（之前就有）
  2. 位置/连接表微调（之前就有）
  3. 惊异度引导的加权更新（新）
     ↓
地形被更新 → 下次推理时预测更准
     ↓
误差持续缩小 → 地形稳定 → 思考层「学会了」这片地形的动力学
     ↓
如果某区域误差始终不降 → 自动触发观测 → 主动模式深入理解
```

四种意外触发源并列：
- 注意力熵增 → 观测
- 语义边界 → 观测
- 预测误差持续异常 → 观测
- 首次收敛到高引力球体 → 观测

## 仍未解决的问题

这是实话实说的部分。

**预测编码层面：**

1. **预测头 B 的 N 值** — 场域稳定性预测中，「接下来 N 步」的 N 是多少？太短（N=1）会被噪声淹没，太长（N=100）会丢失细粒度事件。N 本身应该是一个自适应的参数，而不是预设值。

2. **预测头的初始化** — 地形还没积累时，预测头从零开始完全盲目。要不要在训好的行动层上先做一轮监督预热，让地形有一个可用的初值，再从自监督接手？

3. **损失自适应加权的收敛性** — α/β 的比值自适应的动力学——它收敛到一个稳定值，还是在某些场景下会无限震荡？

**架构层面（从 Phase 3 遗留下来的）：**

4. **引力动力学稳定性** — 两个球体靠近 → 引力增强 → 继续靠近 → 会不会爆炸？（当前衰减项在压着，但没有数学证明）

5. **球体生命周期** — 长期不活跃的球体衰减还是归档？合并阈值怎么定？

6. **探测场半径** — 固定值还是动态值？高质量球体影响更远——但控制参数的物理意义？

7. **原型验证** — 小模型 + 外挂思考层，在矛盾文本推理任务上验证引力扰动的实际效果。

## 给这个系列一个定位

这个系列开始的时候，我没有想清楚它到底在做什么。

前几期写的是思考层、行动层、地形、引力——听起来像在改进 Transformer。但我后来意识到，这不是一个通用架构提案。

如果把它拿到 GPT-6 面前，它的组件一个都用不上。双曲空间的工程成本太高，两层的时序解耦跟现有训练框架不兼容，多样性的注意力指纹度量需要模型内部访问权限。

**但它在另一个场景下是自洽的：personal model。**

一个只服务于一个人的模型。它的知识不来自全互联网，来自这个人的文档、对话、浏览记录。它的引力分布不为通用用户分布优化，只收敛到一个人的认知模式上。它的场域权重不是预训练的产物，是这个人亲手告诉它的。

这个场景下，重力空间架构的所有组件都是直接需要的：

- **地形** = 这个人的知识结构映射
- **场域** = 他的人在不同工作模式下关注什么
- **多样性** = 他的信息来源跨度
- **观测** = 他的决策过程回溯
- **预测编码** = 系统对齐他的预期

这不是巧合。这个架构从起点就是为个人化模型设计的——只是我花了四期才意识到。
